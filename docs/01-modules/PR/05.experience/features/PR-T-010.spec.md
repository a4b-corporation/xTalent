# PR-T-010: Audit Log Viewer

**Type**: T (Transaction — read-only with export)
**Priority**: P0
**BC**: BC-07 — Data Integrity and Compliance
**Actors**: PAYROLL_ADMIN (read within own legal entity), PLATFORM_ADMIN (read all)
**Linked Stories**: US-044, US-045
**API Endpoints**:
- `GET /api/v1/payroll-domain/audit-logs` — Query audit log (filterable, paginated)
- `GET /api/v1/payroll-domain/audit-logs/{event_id}` — Get single event detail
- `GET /api/v1/payroll-domain/audit-logs/export` — Export filtered results as CSV

---

## Overview

The Audit Log Viewer provides a searchable, read-only view of every action taken in the payroll system. Audit records are immutable: they cannot be edited or deleted under any circumstances (enforced at the API and database level). Retention is 7 years minimum (some BHXH records require 10 years).

The viewer is the primary tool for compliance investigations, approval trail verification, and security incident response.

---

## Business Rules

- **BR-098**: Audit records are strictly read-only. No DELETE, UPDATE, or PATCH operation is permitted on audit_log records.
- **BR-099**: Retention minimum 7 years from the event date. BHXH-related events retained 10 years.
- Every state change, field update, data export, login, and approval action is logged.
- Audit records for LOCKED period results are retained indefinitely (no expiry).

## Screen Layout

### Filter Bar

All filters are optional; combining them narrows results.

| Filter | Type | Notes |
|--------|------|-------|
| Legal entity | Select | PAYROLL_ADMIN sees only their entity; PLATFORM_ADMIN sees all or can filter |
| Date range | Date range picker | Default: last 30 days |
| Event type | Multi-select | STATE_CHANGE / FIELD_UPDATE / LOGIN / EXPORT / APPROVAL / EXCEPTION_ACKNOWLEDGE / LOCK |
| Aggregate type | Multi-select | PayrollRun / PayrollResult / PayProfile / PayGroup / PayElement / StatutoryRule / PayPeriod / WorkerPayAssignment |
| Aggregate ID | Text input | Filter events for a specific entity by its UUID |
| Actor | Search/select | Filter by user who performed the action; search by name |

[Apply Filters] button. Filter state is preserved in URL (bookmarkable).

[Clear Filters] resets to defaults.

---

### Results Table

Paginated: 100 rows per page (configurable: 25 / 50 / 100).
Sorted: timestamp descending by default (most recent first).
Total results count shown.

| Column | Width | Notes |
|--------|-------|-------|
| Timestamp | Fixed | ISO 8601 with timezone; localized display |
| Actor | Medium | Full name + role badge (e.g., "Nguyen T. Lan — PAYROLL_ADMIN") |
| Event Type | Small | Colored badge |
| Aggregate Type | Small | Entity type |
| Aggregate ID | Medium | Truncated UUID with copy button; links to entity detail if still accessible |
| Summary | Large | Human-readable one-line description (e.g., "PayrollRun PR-2026-03-001 submitted for approval") |
| Before / After | Small | Expand toggle for FIELD_UPDATE events |

Row click: opens the event detail panel (right-side drawer or full-page view).

---

### Event Detail Panel

Shown when a row is clicked or [View Detail] is selected.

**Sections**:

1. **Event metadata**:
   - Event ID (UUID)
   - Timestamp
   - Actor name, email, role
   - IP address (for LOGIN events)
   - Legal entity

2. **Event data**:
   - Event type badge
   - Aggregate type + aggregate ID
   - Human-readable summary

3. **Before / After JSON diff** (FIELD_UPDATE events only):
   - Side-by-side diff view: "Before" (left) and "After" (right) JSON
   - Changed fields highlighted (using JSON diff library conventions: additions green, removals red)
   - [Copy JSON] button on each side

4. **Related events**: list of other events with the same aggregate_id (to trace the full lifecycle of an entity).

---

### Audit Log Export

[Export CSV] button: exports all events matching the current filters.

**Size limit**: up to 100,000 rows per export. If the filtered result set is larger:
- Warning: "Your filter returns [N] rows. Export is limited to 100,000 rows. Narrow your filters or contact Platform Admin for a full data export."

CSV columns: event_id, timestamp, actor_name, actor_email, actor_role, event_type, aggregate_type, aggregate_id, summary, before_json, after_json, legal_entity_id, ip_address.

File name: `audit_log_{legal_entity}_{date_from}_{date_to}_{exported_at}.csv`

Download is async for large exports: "Your export is being prepared. You will receive an in-app notification when it is ready to download." Max preparation time: 2 minutes.

## Interaction Points

| Interaction | Point |
|-------------|-------|
| Browse audit log | Payroll > Compliance > Audit Log |
| Filter by entity | Enter aggregate_id (often accessed from the entity detail page via "View Audit Trail" link) |
| View field change diff | Event detail panel for FIELD_UPDATE events |
| Export for compliance review | [Export CSV] button |
| Access from entity context | "View Audit Trail" link on PayrollRun, PayProfile, StatutoryRule detail pages opens pre-filtered audit log |

**Deep link from entity pages**: major entity detail pages (PayrollRun, PayProfile, StatutoryRule, PayGroup) include a [View Audit Trail] button that links to the audit log pre-filtered to `aggregate_type=X&aggregate_id={id}`.

## Performance Notes

Audit logs accumulate rapidly (every payroll action writes one or more records). The viewer must:
- Never scan the full table without an index. All queries are index-backed by: `(legal_entity_id, timestamp)`, `(aggregate_type, aggregate_id)`, `(actor_id, timestamp)`.
- Default 30-day window protects against accidental full-table scans.
- Result count is estimated for large result sets (> 10,000 rows); exact count shown after filtering narrows to < 10,000 rows.

## Error Handling

| Scenario | Behavior |
|----------|---------|
| Actor attempts to delete/modify audit record | HTTP 405 Method Not Allowed at API level; no delete UI exists |
| PAYROLL_ADMIN queries another legal entity's logs | HTTP 403; filter silently scoped to own legal entity |
| Date range > 365 days | Warning: "Date ranges over 365 days may return a large result set and slow down the query. Consider narrowing the range or using export." Query still executes. |
| Export > 100,000 rows | Warning with option to narrow filters; export capped at 100,000 rows with a note in the CSV header |
| Event detail missing before/after (older format) | "Before/after data not available for events prior to [date of schema migration]." |
