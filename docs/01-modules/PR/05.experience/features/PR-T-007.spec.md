# PR-T-007: Payslip Generation

**Type**: T (Transaction)
**Priority**: P0
**BC**: BC-04 — Payroll Output and Disbursement
**Actors**: PAYROLL_ADMIN (generate and distribute)
**Linked Stories**: US-031
**API Endpoints**:
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/payslips/generate` — Initiate bulk generation
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/payslips/status` — Generation job status
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/payslips/{worker_id}/preview` — Single payslip preview
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/payslips/download` — Download all as ZIP
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/payslips/{worker_id}/download` — Download single PDF
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/payslips/mark-distributed` — Mark as distributed

---

## Overview

After a payroll run is APPROVED or LOCKED, the Payroll Admin can bulk-generate PDF payslips for all workers in the run. In V1, distribution is manual (admin downloads a ZIP and distributes). In V2, payslips will be auto-sent to worker email or made available in the worker self-service portal automatically.

---

## Pre-Conditions

- Payroll run status = APPROVED or LOCKED
- Run type = PRODUCTION (DRY_RUN and SIMULATION do not generate distributable payslips)

---

## Workflow Steps

### Step 1: Select Run

The admin navigates to a run detail page and clicks the [Generate Payslips] button from the "Payslips" tab.

The system shows:
- Run reference: Pay Group, Period, Run ID
- Worker count: [N] workers in this run
- Prior generation: if payslips were generated before, show "Last generated: [date] by [admin]. [N] PDFs. [Regenerate All]"

[Generate All Payslips] button initiates bulk generation.

---

### Step 2: Generate (Async)

On clicking [Generate All Payslips]:
- Async job starts on the server.
- Progress indicator: "Generating payslips... [N] / [Total] completed."
- Admin can navigate away. A persistent banner appears on other pages: "Payslip generation in progress for [Pay Group] [Period]."
- On completion: toast notification and the Payslips tab updates with download options.

---

### Step 3: Preview Single Payslip

Before or after bulk generation, admin can preview any single worker's payslip:

[Preview] button in the worker result table (on run detail or in the Payslips tab).

Opens a PDF preview in a modal or new browser tab.

---

### Step 4: Download

**Download all as ZIP**:
- [Download All (ZIP)] button on the Payslips tab.
- ZIP file name: `payslips_{pay_group}_{period_yyyymm}_{generated_at}.zip`
- Each PDF inside: `{employee_id}_{worker_last_name}_{period_yyyymm}.pdf`

**Download single PDF**:
- Per-row [Download] button in the payslips worker table.

---

### Step 5: Mark as Distributed

After the admin has delivered payslips to workers (via email, portal, or physical copy), they mark distribution complete:

[Mark as Distributed] button on the Payslips tab.

Confirmation modal:
- "Confirm that payslips for [Pay Group] [Period] have been distributed to all [N] workers."
- Distribution method: dropdown (PORTAL / EMAIL / PHYSICAL / OTHER)
- Distribution date: defaults to today; editable.
- Note field: optional, 200 chars.

On confirmation: payslip distribution status → DISTRIBUTED. Recorded in audit log.

**V2 note**: Reserved for auto-send via email or push notification to worker portal. Email template must be designed in V1 (placeholder) even though it is not triggered automatically.

## Payslip PDF Content

The payslip is a single-page PDF. Layout:

**Header section**:
- Company logo (from legal entity configuration)
- Company name and address
- Tax code
- Title: "PHIẾU LƯƠNG" (Payroll Slip) — bilingual if configured
- Period: Tháng [MM] / [YYYY]

**Worker section**:
- Full name
- Employee ID
- Department
- Position

## Error Handling

| Scenario | Behavior |
|----------|---------|
| Generation attempted on DRY_RUN | Button disabled: "Payslips are only generated for PRODUCTION runs." |
| Generation attempted before APPROVED status | Button disabled: "Payslip generation requires an approved run." |
| Single worker PDF generation fails | Worker row shows error badge; [Retry Single] button available. Bulk download still works for all successful PDFs. |
| Bulk ZIP download times out (large run) | After 30 seconds: "Your download is taking longer than expected. We will notify you when it is ready." Download link emailed to admin (V1: fallback to chunked download). |
| Company logo not configured | Payslip renders without logo; advisory on settings page: "Add a company logo in Legal Entity settings to include it on payslips." |
