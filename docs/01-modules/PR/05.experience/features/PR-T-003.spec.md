# PR-T-003: Payroll Run Approval

**Type**: T (Transaction)
**Priority**: P0
**BC**: BC-03 — Payroll Execution Engine
**Actors**: HR_MANAGER (Level 2), FINANCE_MANAGER (Level 3)
**Linked Stories**: US-025, US-026, US-027
**API Endpoints**:
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/approval` — Get approval state
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/approval/level2/approve` — HR Manager approve
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/approval/level2/reject` — HR Manager reject
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/approval/level3/approve` — Finance Manager approve
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/approval/level3/reject` — Finance Manager reject

---

## Overview

After a PRODUCTION payroll run has all exceptions acknowledged, the Payroll Admin submits it for two-level approval. Level 2 is HR Manager; Level 3 is Finance Manager. Both levels must approve in sequence before the run can be locked. Either level can reject, which returns the run to the Payroll Admin for correction or re-submission.

The approval screen is the same layout for both approver roles; only the available actions differ based on the actor's role and the current approval state.

---

## Approval State Machine

```
PENDING_APPROVAL
  → Level 2 Approves → PENDING_L3_APPROVAL
  → Level 2 Rejects  → REJECTED (returns to Payroll Admin)

PENDING_L3_APPROVAL
  → Level 3 Approves → APPROVED
  → Level 3 Rejects  → REJECTED (returns to Payroll Admin)
```

- A Finance Manager cannot act on a run in PENDING_APPROVAL status (Level 2 not yet complete).
- An HR Manager cannot act on a run in PENDING_L3_APPROVAL status (their step is already complete).
- Self-approval is not possible: the Payroll Admin who submitted cannot also be an approver (even if they hold multiple roles).

## Notification Flow

| Event | Notification Recipients | Channel |
|-------|------------------------|---------|
| Run submitted (PENDING_APPROVAL) | All HR Managers for the legal entity | In-app + email |
| Level 2 approved (PENDING_L3_APPROVAL) | All Finance Managers for the legal entity | In-app + email |
| Level 2 rejected | Payroll Admin who submitted the run | In-app + email |
| Level 3 approved (APPROVED) | Payroll Admin + HR Managers | In-app |
| Level 3 rejected | Payroll Admin who submitted the run | In-app + email |

Email template: subject includes Pay Group name, period, and action. Body includes run summary totals and the approval comment.

## Approval Screen Layout

**Access**: Payroll > Approvals (HR Manager and Finance Manager landing); or direct link from notification.

### Header

- Run reference: Run ID, Pay Group, Period, Run Mode
- Status badge: PENDING_APPROVAL / PENDING_L3_APPROVAL
- Submitted by: Payroll Admin name and timestamp
- Submission note (if provided by Payroll Admin)

### Run Summary (read-only)

Same summary cards as PR-T-001 Step 5:
- Total Gross, Total SI (Employee), Total SI (Employer), Total PIT, Total Net, Worker Count
- Period-over-period variance table (amber highlights for variances > threshold)

### Exception Summary (read-only)

If any exceptions were acknowledged:
- Table: exception type, count, resolution applied
- No open exceptions should exist at this stage (enforced by submission pre-condition)

### Approval History

Shows the approval trail:

| Level | Role | Action | Actor | Timestamp | Comment |
|-------|------|--------|-------|-----------|---------|
| Level 2 | HR Manager | APPROVED | Nguyen T. Lan | 2026-03-15 14:22 | LGTM |
| Level 3 | Finance Manager | PENDING | — | — | — |

Empty rows for levels not yet acted on.

### Approver Actions

Only visible to the approver whose turn it is:

**Approve**:
- [Approve] button.
- Confirmation modal: "You are approving the payroll run for [Pay Group] [Period]. This action is recorded in the audit log."
- Optional comment field (max 500 chars).
- [Confirm Approval] button in modal.

**Reject**:
- [Reject] button.
- Rejection modal:
  - Reason code (required): dropdown.

| Reason Code | Description |
|-------------|-------------|
| DATA_ERROR | Incorrect worker data detected |
| FORMULA_ISSUE | Formula or rate appears incorrect |
| MISSING_WORKERS | Workers are missing from this run |
| VARIANCE_TOO_HIGH | Unexplained variance from prior period |
| INCOMPLETE_EXCEPTIONS | Exceptions not properly resolved |
| OTHER | Requires free-text explanation |

  - Comment field: required when reason = OTHER; optional otherwise; max 500 chars.
  - [Confirm Rejection] button.

## Rejection Handling (Payroll Admin View)

When a run is rejected, the Payroll Admin receives a notification. On the run detail page:

- Status badge: REJECTED (red)
- Rejection panel:
  - Rejected by: [Role] [Name]
  - Rejected at: [Timestamp]
  - Reason: [reason_code] — [comment]

**Available actions**:

1. **Re-submit same run** (if correction is minor and data issue was external):
   - Payroll Admin can add a response comment and re-submit without re-running the computation.
   - Allowed only if the rejection reason does not require recalculation (e.g., DATA_ERROR or FORMULA_ISSUE would require a new run).
   - System shows advisory: "Re-submitting the same run without recalculation. Ensure the issue has been resolved externally."

2. **Initiate new run**:
   - [New Run] button: starts PR-T-001 wizard with the same pay_group + period pre-filled.
   - The rejected run remains in history with status = REJECTED.

## Business Rules

| Rule | Description |
|------|-------------|
| BR-082 | Two-level approval is mandatory for all PRODUCTION runs |
| BR-083 | Level 3 (Finance Manager) cannot approve before Level 2 (HR Manager) |
| BR-084 | Approver cannot be the same person who submitted the run |
| BR-085 | All rejection reasons are stored in the audit log |

---

## Error Handling

| Scenario | Behavior |
|----------|---------|
| Finance Manager attempts to approve at Level 2 stage | [Approve] button not shown; status message: "Awaiting Level 2 approval from HR Manager." |
| HR Manager attempts to act at Level 3 stage | [Approve] / [Reject] buttons not shown; status message: "Level 2 approval complete. Awaiting Finance Manager approval." |
| Payroll Admin attempts to approve their own submission | Approval buttons not rendered; message: "You submitted this run. A different approver must review it." |
| Run modified after submission (should not be possible by design) | System enforces PENDING_APPROVAL run immutability at API level; any attempt returns HTTP 409 |
