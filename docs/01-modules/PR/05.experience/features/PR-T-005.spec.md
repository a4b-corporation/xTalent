# PR-T-005: Termination Payroll

**Type**: T (Transaction)
**Priority**: P0
**BC**: BC-04 — Payroll Output and Disbursement
**Actors**: PAYROLL_ADMIN (primary), HR_MANAGER (may initiate or review)
**Linked Stories**: US-019
**API Endpoints**:
- `POST /api/v1/payroll-usecase/payroll-runs` — Create run with run_type=TERMINATION
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}` — Get run detail
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/termination-preview` — Proration and element preview
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/submit-for-approval` — Submit for approval

---

## Overview

A Termination Payroll is an off-cycle run for a single worker who is leaving the organization. It computes a pro-rated final salary, optionally includes unused annual leave payout and severance pay, and generates a PIT certificate if required. The workflow mirrors the monthly run (PR-T-001) but is scoped to one worker and carries termination-specific elements.

**Trigger modes**:
1. **Automatic** (preferred): triggered when a `WorkerTerminated` domain event is received from the Core Organization (CO) module. A termination payroll task is auto-created and surfaced as a notification to the Payroll Admin.
2. **Manual**: Payroll Admin initiates directly from the Payroll Runs screen for cases where the CO event is delayed or for retroactive termination processing.

---

## Trigger: Automatic Initiation

On receiving `WorkerTerminated` event:

1. System creates a Termination Payroll task (status: PENDING_INITIATION).
2. Payroll Admin receives in-app notification: "Termination payroll required for [Worker Name] (effective [termination_date]). [Start Termination Payroll]"
3. Admin clicks the notification to enter the setup wizard pre-filled with the worker's details.

---

## Wizard Steps

### Step 1: Setup

Pre-filled fields (from WorkerTerminated event or manual lookup):

| Field | Source | Editable |
|-------|--------|---------|
| working_relationship_id | CO event / worker search | No (auto-triggered) / Yes (manual) |
| Worker name | Derived from CO | Read-only |
| Termination date | CO event | Read-only if auto; editable if manual |
| Termination reason | CO event (enum: RESIGNATION / MUTUAL_AGREEMENT / DISMISSAL / CONTRACT_END / RETIREMENT / DEATH) | Read-only if auto |
| Pay group | Current assignment | Read-only |
| Pay profile | Current assignment | Read-only |
| Pay period | Derived: period containing termination_date | Read-only |
| Statutory rule effective date | Defaults to termination_date | Editable |

Manual initiation additionally shows a [Search Worker] field to locate the working_relationship_id.

[Preview Final Pay] button: triggers server-side calculation preview (Step 2).

---

### Step 2: Element Selection and Preview

After clicking [Preview Final Pay], the system computes a proration preview and presents it.

**Proration Preview Panel** (read-only):

| Field | Value |
|-------|-------|
| Period start | [date] |
| Period end | [date] |
| Termination date | [date] |
| Proration method | [CALENDAR_DAYS / WORK_DAYS] (from pay profile) |
| Work days in period | [N] |
| Entitled days | [N] (days from period start to termination date inclusive) |
| Proration factor | [N] / [N] = [0.xx] |
| Base salary | [amount] VND |
| Pro-rated base salary | [amount × factor] VND |

**Element Checklist**: admin selects which optional elements to include in the final pay:

| Element | Default | Notes |
|---------|---------|-------|
| Base salary (pro-rated) | Included (mandatory) | Cannot be deselected |
| Unused annual leave payout | Checked | Requires `unused_leave_days` input (pre-filled from CO if available; editable) |
| Severance pay | Conditional | Visible only for DISMISSAL / MUTUAL_AGREEMENT / REDUNDANCY termination reasons per labor law |
| Notice period pay | Conditional | Visible if termination_reason = DISMISSAL and notice was not served |

For each selected optional element, the system shows the calculated amount using the current statutory rules. Admin can override the calculated amount with a justification note.

**PIT Certificate**:
[ ] Auto-generate PIT withholding certificate (Form 03/TNCN) after run completion.
Default: checked.
This will trigger bulk certificate generation for this one worker after the run is LOCKED.

**Standard deductions** are applied automatically and shown read-only:
- SI (employee share)
- PIT withheld (using final month's accumulated YTD amounts)
- Any prior-period advance recoveries outstanding

[Recalculate] button: re-triggers the preview with updated element selections.

---

## Post-Approval

After Level 3 approval:

- Termination payroll run status → APPROVED.
- [Lock] and [Generate Bank File] actions become available (same as monthly run flow).
- If "Auto-generate PIT certificate" was checked: system queues Form 03/TNCN generation for this worker.
- Worker's pay assignment is ended effective termination_date (system action, not manual).

---

## Business Rules

| Rule | Description |
|------|-------------|
| BR-002 | Proration: CALENDAR_DAYS = entitled_days / calendar_days_in_month |
| BR-003 | Proration: WORK_DAYS = entitled_days / standard_work_days_per_month |
| BR-048 | Severance pay is exempt from PIT up to a statutory cap; excess is taxable |
| BR-010 | Unused leave payout capped at maximum accrued balance from HR module |
| — | Termination reason drives which optional elements are eligible for inclusion |

---

## Error Handling

| Scenario | Behavior |
|----------|---------|
| Worker not yet terminated in CO | Manual initiation allowed with warning: "No WorkerTerminated event received from Core Organization. Ensure CO is updated after submission." |
| Worker has no active pay assignment | Setup wizard shows error: "Worker [name] has no active pay group assignment. Assign the worker to a pay group before initiating termination payroll." |
| Termination date is in a LOCKED period | Blocked: "The period containing [termination_date] is already locked. Contact Platform Admin for a correction run." |
| Unused leave days exceed CO leave balance | Warning (not blocking): "Entered unused leave days [N] exceeds the CO leave balance of [M] days. Confirm with HR before proceeding." |
| Duplicate termination payroll for same worker + period | System detects and blocks: "A termination payroll for [worker] in [period] already exists (run_id: [X])." |
