# PR-T-009: PIT Declaration Export

**Type**: T (Transaction)
**Priority**: P0
**BC**: BC-05 — Statutory Reporting
**Actors**: PAYROLL_ADMIN (primary)
**Linked Stories**: US-034
**API Endpoints**:
- `POST /api/v1/payroll-usecase/statutory-reports/pit-declaration` — Initiate PIT declaration generation
- `GET /api/v1/payroll-usecase/statutory-reports/pit-declaration/{report_id}` — Get report status
- `GET /api/v1/payroll-usecase/statutory-reports/pit-declaration/{report_id}/preview` — Preview data
- `GET /api/v1/payroll-usecase/statutory-reports/pit-declaration/{report_id}/download` — Download XML
- `POST /api/v1/payroll-usecase/statutory-reports/pit-declaration/{report_id}/mark-filed` — Mark as filed

---

## Overview

The PIT Declaration Export generates Form 05/KK-TNCN, the quarterly PIT withholding declaration required for submission to the General Department of Taxation (GDT) via the eTax portal. This form summarizes total PIT withheld per worker for the quarter. The output is an XML file conforming to Circular 08/2013/TT-BTC (and amendments).

Quarterly filing deadlines: Q1 by April 30, Q2 by July 31, Q3 by October 31, Q4 by January 31 of the following year.

---

## Pre-Conditions

- All monthly payroll runs for the selected quarter must have status APPROVED or LOCKED. The system does not require all three months to be locked — it uses whatever runs are available — but warns if a month is missing.
- Approved/locked runs must exist for the legal entity.

## Workflow Steps

### Step 1: Select Parameters

Fields:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| legal_entity_id | Select | Yes | Default: current context |
| year | Select | Yes | Current year default |
| quarter | Select | Yes | Q1 / Q2 / Q3 / Q4 |

On selecting year + quarter, system checks for source runs:

| Month | Run Found | Status |
|-------|-----------|--------|
| Month 1 | Yes | LOCKED |
| Month 2 | Yes | APPROVED |
| Month 3 | No | WARNING: No approved run found |

If a month is missing: warning banner: "PIT data for [month] is not yet available. The declaration will be generated without data for that month. Ensure all monthly runs are complete before filing."

Admin may proceed despite warnings.

[Generate Preview] proceeds to Step 2.

---

### Step 2: Preview Report Data

Displays aggregated PIT data per worker for the quarter.

**Summary header**:
| Metric | Value |
|--------|-------|
| Quarter | Q[N] / [YYYY] |
| Legal Entity | |
| Total workers with PIT | [N] |
| Total PIT withheld | [amount] VND |
| Of which: resident income (type 01) | [amount] VND |
| Of which: non-resident income (type 02) | [amount] VND |

**Worker detail table**:

| Column | Notes |
|--------|-------|
| No. | Sequential |
| Worker Full Name | |
| Tax ID (MST) | Worker's personal tax code |
| Income Type | 01 (resident) / 02 (non-resident) |
| Total Gross Income (quarter) | Sum of taxable gross across all periods |
| PIT Exemptions (quarter) | Sum of personal + dependant deductions applied |
| Taxable Income | Gross minus exemptions |
| PIT Withheld (quarter) | Sum of pit_withheld across all periods |
| Months with Income | Count of months worker had income this quarter |

**Income type breakdown**: workers are grouped by income type (01/02) per Form 05/KK-TNCN structure.

Workers with zero PIT withheld for the quarter are excluded from the XML by default. Toggle: [Include Zero-PIT Workers] to add them (some entities prefer to declare all workers).

Pagination: 50 rows per page.
Search: worker name or tax ID.
Export: [Download Preview CSV].

---

### Step 3: Validate

[Validate Declaration] runs pre-filing checks:

| Check | Description |
|-------|-------------|
| Tax ID coverage | All workers with PIT > 0 must have a tax ID (MST) |
| Legal entity tax code | Legal entity's tax code must be configured in system |
| Income type classification | All workers have an income type assigned |
| PIT bracket consistency | PIT withheld per worker is consistent with applicable PIT_BRACKET rule |

Workers missing tax ID: listed in a warning panel. These workers will be excluded from the XML (required field). Admin must update tax ID in CO module.

[Generate XML] button enabled when validation passes (or admin explicitly proceeds with warnings).

---

### Step 4: Generate XML

[Generate XML] button: async generation.

**Generated XML structure** (Form 05/KK-TNCN per Circular 08/2013/TT-BTC):
- Declaration header: legal entity name, tax code, tax authority, quarter, year, submission date, declaration type (QUARTERLY)
- Section I: Summary totals (total income, total PIT, worker counts by income type)
- Appendix (Phụ lục): per-worker rows with income and PIT amounts
- Digital signature placeholder (not implemented in V1; admin signs via GDT eTax portal)

File name: `05_KKTNCN_{legal_entity_tax_code}_{year}_Q{quarter}_{generated_at}.xml`

---

### Step 5: Download

[Download XML] button on READY report.

The file is archived in the Statutory Reports history.

---

### Step 6: Mark as Filed

After uploading to the GDT eTax portal:

[Mark as Filed] button.

Confirmation modal:
- Filed date: defaults to today; editable.
- GDT reference number: optional.
- Filing channel: dropdown (ETAX_PORTAL / TAX_OFFICE_DIRECT / AGENT)

On confirmation: report status → FILED. Audit log entry created.

## Business Rules

| Rule | Description |
|------|-------------|
| BR-040 | PIT calculated using progressive brackets from active PIT_BRACKET statutory rule |
| BR-041 | No hardcoded tax rates; formulas must reference statutory_rule records |
| BR-042 | Personal deduction: 11,000,000 VND/month (from statutory rule) |
| BR-043 | Dependant deduction: 4,400,000 VND/dependant/month (from statutory rule) |
| BR-044 | TAX_EXEMPT elements excluded from PIT_BASIS |
| BR-045 | TAX_EXEMPT_CAPPED elements exempt up to cap; excess included in PIT_BASIS |
| BR-048 | Severance pay exempt from PIT up to statutory cap; excess taxable |
| BR-051 | Non-resident workers (income type 02) subject to flat 20% PIT; no bracket calculation |

## Error Handling

| Scenario | Behavior |
|----------|---------|
| Missing source run for a quarter month | Warning shown; admin may proceed; declaration omits that month's data |
| Worker missing tax ID | Excluded from XML; listed in warning panel |
| Legal entity tax code not configured | Validation FAIL; message: "Legal entity [X] has no tax code configured. Configure it in Legal Entity settings before generating PIT declarations." |
| XML generation fails | Error on report record; [Retry] button available |
| Duplicate declaration for same quarter | Warning: "A PIT declaration for Q[N] [YYYY] was already generated on [date]. Generate a new one only if the previous is void." [Proceed] button requires explicit confirmation |
