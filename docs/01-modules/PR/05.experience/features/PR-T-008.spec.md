**Type**: T (Transaction)

**Actors**: PAYROLL_ADMIN (primary)

**API Endpoints**:
- `POST /api/v1/payroll-usecase/statutory-reports/bhxh` — Initiate BHXH report generation
- `GET /api/v1/payroll-usecase/statutory-reports/bhxh/{report_id}` — Get report status

---

## Overview

The BHXH Report Export generates the D02-LT monthly declaration form required by the Vietnam Social Insurance Agency (BHXH). This form reports SI contribution amounts (employer and employee shares) for every SI-eligible worker. The output is an XML file in VssID format, which the admin uploads manually to the VssID portal.

This report must be filed monthly, typically by the 15th of the following month.

---

## Pre-Conditions

- At least one payroll run for the selected legal entity + period must have status APPROVED or LOCKED.
- Workers must have SI eligibility resolved (no ZERO_BHXH_SI_ELIGIBLE open exceptions in the source run).

## Workflow Steps

### Step 1: Select Parameters

Fields:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| legal_entity_id | Select | Yes | Default: current context |
| period | Select (month/year) | Yes | Must have an approved/locked run |
| source_run_id | Select | Yes | Shows approved/locked runs for the selected entity + period; defaults to the most recent PRODUCTION run |

On selecting the source run, system shows a read-only summary:
- Payroll run date: [date]
- SI-eligible workers: [N]
- Workers excluded from SI: [N] (contract-exempt)
- Total employer SI: [amount] VND
- Total employee SI: [amount] VND

[Generate Preview] button: proceeds to Step 2.

---

### Step 2: Preview Report Data

Displays the D02-LT data table for review before generating the official XML:

| Column | Notes |
|--------|-------|
| No. | Sequential |
| Worker Full Name | |
| CMND / CCCD | National ID |
| SI Code (BHXH code) | Worker's registered SI number |
| SI Salary | Monthly SI basis for this worker |
| BHXH Employer | Employer BHXH contribution (17.5% × SI salary) |
| BHYT Employer | Employer BHYT contribution (3% × SI salary) |
| BHTN Employer | Employer BHTN contribution (1% × SI salary) |
| BHXH Employee | Employee BHXH deduction (8%) |
| BHYT Employee | Employee BHYT deduction (1.5%) |
| BHTN Employee | Employee BHTN deduction (1%) |
| Total Employer | Sum of employer contributions |
| Total Employee | Sum of employee deductions |

Summary totals row at bottom of table.

**Validation alerts** (shown above the table):

| Alert | Condition |
|-------|-----------|
| Warning | Worker has SI code = NULL. SI code is required in the XML. These workers are excluded. Count shown. |
| Warning | SI salary exceeds ceiling for [N] workers. Capped values used. |
| Error | [N] workers have unresolved ZERO_BHXH_SI_ELIGIBLE exceptions. Report will not include these workers. |

Admin can proceed if no errors. Workers with SI code = NULL are excluded with a downloadable exclusion list.

Pagination: 50 rows per page.
Search: worker name or SI code.
Export preview: [Download Preview CSV] before generating the XML.

---

### Step 3: Validate

[Validate Report] button runs a final pre-submission check:

| Check | Description |
|-------|-------------|
| All workers have SI code | Required field for D02-LT XML |
| Employer rates match active SI_RATE rule | Cross-checks calculated rates vs statutory_rule |
| Employee rates match active SI_RATE rule | Same |
| SI salary ceiling applied where required | Workers over ceiling have capped amounts |

Results: PASS (green) or FAIL (red) per check, with detail message.

If any check FAILS: [Generate XML] button is disabled until resolved.
If all PASS: [Generate XML] button enabled.

---

### Step 4: Generate XML

[Generate XML] button: async generation.

Progress: "Generating D02-LT XML..." spinner.

On completion: report status → READY. Download button appears.

**Generated XML structure** (D02-LT VssID format):
- Root element: `<BHXH_D02_LT>`
- Header: legal entity name, tax code, period, submission date, total workers, total employer amount, total employee amount
- Worker records: one `<NLD>` element per worker with all required fields
- Checksums per VssID specification

---

### Step 5: Download

[Download XML] button.

File name: `D02-LT_{legal_entity_code}_{period_yyyymm}_{generated_at}.xml`

The file is also archived in the Statutory Reports history (accessible from Payroll > Statutory Reports).

---

### Step 6: Mark as Submitted

After uploading to the VssID portal:

[Mark as Submitted] button.

Confirmation modal:
- Submission date: defaults to today; editable.
- VssID submission reference number: optional text field; max 100 chars.

On confirmation: report status → SUBMITTED. Recorded in audit log.

## Business Rules

| Rule | Description |
|------|-------------|
| BR-020 | BHXH employer rate: 17.5% of SI_BASIS (capped at monthly ceiling) |
| BR-021 | BHYT employer rate: 3.0% of SI_BASIS |
| BR-022 | BHTN employer rate: 1.0% of SI_BASIS |
| BR-023 | BHXH employee rate: 8.0% of SI_BASIS |
| BR-024 | BHYT employee rate: 1.5% of SI_BASIS |
| BR-025 | BHTN employee rate: 1.0% of SI_BASIS |
| BR-028 | SI monthly ceiling applied per active SI_RATE statutory rule |

---

## Error Handling

| Scenario | Behavior |
|----------|---------|
| Source run not yet approved | Period selector shows warning: "No approved run found for [period]. Approve a payroll run first." |
| Worker missing SI code | Worker excluded from XML; listed in exclusion warning; downloadable list |
| Rate mismatch vs statutory rule | Validation FAIL with detail: "BHXH employer rate in results is 17.0% but current SI_RATE rule specifies 17.5%. Rerun payroll with the correct statutory rule effective date." |
| XML generation fails | Error on report record; [Retry] button available |
