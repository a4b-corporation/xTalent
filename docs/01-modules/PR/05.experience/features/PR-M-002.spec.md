## Overview

Pay Elements are the atomic building blocks of payroll: individual earnings, deductions, or employer contributions. Each element carries a formula (or fixed value), tax treatment, SI basis inclusion, and proration method. Formulas must reference statutory_rule records for all rate values — literal numeric rates are rejected at compile time.

## Business Rules

- **BR-027**: SI basis inclusion — each element independently configures whether it is included in SI_BASIS.
- **BR-041**: No hardcoded statutory rates — formulas must reference statutory_rule records. Literals produce a compile-time warning requiring Finance Lead sign-off.
- **BR-044/045/046**: Tax treatment options: TAXABLE / TAX_EXEMPT / TAX_EXEMPT_CAPPED. TAX_EXEMPT_CAPPED elements reference a statutory_rule for the cap amount.
- **BR-003**: Elements carry their own proration_method (CALENDAR_DAYS / WORK_DAYS / NONE). Element-level setting overrides pay profile default.
- Formula lifecycle: DRAFT → PENDING_APPROVAL → APPROVED → ACTIVE → DEPRECATED. Only ACTIVE formulas are used in payroll calculations.
- A deprecated formula remains readable for audit; no new payroll run uses it.
- Element code must be unique per country_code.

### Formula Editor
- Inline formula text editor with syntax highlighting
- [Validate Formula] button — runs formula compiler:
  - Checks referenced functions and variables exist
  - Detects literal numeric rate values → warning: "Literal numeric rates found. Recommend referencing statutory_rule() instead. Requires Finance Lead sign-off to activate."
  - References to `statutory_rule('VN_BHXH_EMP_RATE')` pass validation cleanly
- Formula test runner (optional): enter sample input values → see computed output

### Formula Lifecycle States

```
DRAFT
  ↓ [Submit for Approval]
PENDING_APPROVAL
  ↓ [Finance Lead Approves]
APPROVED
  ↓ [Payroll Admin Activates]
ACTIVE
  ↓ [Deprecate] (when new version activated)
DEPRECATED
```

### Validation Rules

- **Duplicate code**: "Element code must be unique for this country."
- **Literal rate rejected**: "Statutory rates must reference statutory_rule records. Found literal: 0.08. Submit for Finance Lead approval to proceed."
- **Missing cap rule**: "Tax treatment TAX_EXEMPT_CAPPED requires a statutory rule reference for the exempt cap."
- **Cannot activate without approval**: "Element must be APPROVED before activation. Submit for Finance Lead review."
- **Edit blocked after ACTIVE**: "Active element cannot be edited. Deprecate this version and create a new one."
