# PR-A-002: Payroll Variance Report

**Type**: A — Analytics
**Priority**: P0
**BC**: BC-03 Payroll Execution
**Actors**: PAYROLL_ADMIN, FINANCE_MANAGER
**Linked Stories**: US-037
**API Endpoints**: `GET /payroll/reports/variance`

---

## Overview

Period-over-period payroll variance analysis. Highlights workers with significant changes in gross pay, helping Payroll Admin and Finance Manager identify anomalies before finalising approvals.

---

## Data Sources

| Source Table | Usage |
|-------------|-------|
| `payroll_result` | Current and previous period gross/SI/PIT/net amounts |
| `compensation_snapshot` | pay_group, pay_method for context |
| `retroactive_adjustment` | Flag rows where variance is due to retro adjustment |

---

## Metrics

| Metric | Definition |
|--------|-----------|
| Current Gross | payroll_result.gross_amount for selected period |
| Previous Gross | payroll_result.gross_amount for comparison period |
| Variance Amount | Current Gross − Previous Gross |
| Variance % | (Variance Amount / Previous Gross) × 100 |
| Net Variance | Current Net − Previous Net |
| Variance Source | "RETRO", "PAY_CHANGE", "PRORATION", "NEW_HIRE", "TERMINATION", "NORMAL" |

---

## Variance Source Classification

| Condition | Label |
|-----------|-------|
| RetroactiveAdjustment exists for this worker in current period | RETRO |
| Worker not in previous period (new_hire or first run) | NEW_HIRE |
| Worker not in current period (terminated) | TERMINATION |
| Gross variance > ±20% with no retro | SIGNIFICANT |
| All other | NORMAL |

---

## Filters

- **Legal Entity**: from context switcher
- **Current Period**: month/year picker
- **Comparison Period**: defaults to previous month; can select any past period
- **Pay Group**: multi-select
- **Variance Threshold**: slider (default ±20%); show only rows exceeding threshold
- **Variance Source**: filter by label (RETRO, NEW_HIRE, TERMINATION, SIGNIFICANT, NORMAL)

---

## Visualization

1. **Summary Cards**: New Hires | Terminations | Workers with >±20% variance | Workers with retro adjustment
2. **Scatter Plot** (optional): X=Previous Gross, Y=Current Gross; diagonal = no change; outliers visible
3. **Detail Table** (primary): sortable by Variance %, flagged rows highlighted

### Detail Table Columns

| Column | Description |
|--------|-------------|
| Worker Name | From CO via snapshot |
| Pay Group | |
| Pay Method | |
| Previous Gross | VND |
| Current Gross | VND |
| Variance Amount | VND (red if negative, green if positive) |
| Variance % | Bold if > threshold |
| Net Variance | VND |
| Source | Badge: RETRO / NEW_HIRE / TERMINATION / SIGNIFICANT / NORMAL |
| Reason | Retro adjustment description if RETRO; blank otherwise |

---

## Alert Threshold

Default alert threshold: ±20% variance. Configurable per legal entity in settings. Rows exceeding threshold are highlighted in amber. Rows with RETRO label show info icon explaining the retro source.
