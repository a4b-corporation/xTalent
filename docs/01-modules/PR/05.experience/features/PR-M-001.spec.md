# PR-M-001: Pay Group Management

**Type**: M (Masterdata)
**Priority**: P0
**BC**: BC-01 — Pay Master Configuration
**Actors**: PAYROLL_ADMIN (primary), HR_MANAGER (read), FINANCE_MANAGER (read)
**Linked Stories**: US-001
**API Endpoints**:
- `POST /api/v1/payroll-domain/pay-groups` — Create pay group
- `GET /api/v1/payroll-domain/pay-groups` — List pay groups
- `GET /api/v1/payroll-domain/pay-groups/{id}` — Get pay group detail
- `PUT /api/v1/payroll-domain/pay-groups/{id}` — Update pay group
- `POST /api/v1/payroll-domain/pay-groups/{id}/deactivate` — Deactivate pay group

---

## Overview

Pay Groups are the top-level organizational unit for payroll execution. Each PayGroup belongs to a single legal entity, uses one pay calendar, and defines the currency for all payroll runs. Workers are enrolled into pay groups; payroll runs are executed against a pay group + period combination.

## Business Rules

- **BR-101**: A PayGroup must be bound to exactly one legal entity. A PayGroup from Legal Entity A is never visible to Payroll Admins of Legal Entity B.
- **BR-100**: Data isolation — cross-legal-entity access returns HTTP 403.
- A PayGroup code must be unique within a legal entity (system-wide uniqueness not required).
- A PayGroup cannot be deleted if it has LOCKED periods. Deactivation is the only allowed action.
- A deactivated PayGroup can be reactivated if no conflicting active PayGroup has the same code.
- Only one non-LOCKED pay period per PayGroup is permitted at a time.

## CRUD Specification

### Fields

| Field | Type | Required | Validation | Notes |
|-------|------|----------|------------|-------|
| code | String | Yes | Unique within legal_entity; alphanumeric + hyphens; max 50 chars | Immutable after creation |
| name | String | Yes | Max 200 chars | Display name |
| legal_entity_id | UUID | Yes | Must exist in org_legal.entity | Immutable after creation; drives data scoping |
| pay_calendar_id | UUID | Yes | Must exist and belong to same legal entity | Determines period schedule and cut-off dates |
| currency | String | Yes | ISO 4217; VND for Vietnamese entities | Default: VND |
| status | Enum | System | ACTIVE / INACTIVE | Default: ACTIVE on creation |
| description | String | No | Max 500 chars | Free-text notes |
| ot_monthly_cap | Integer | No | Default: 24 (hours) | Configurable per BR-008 |
| standard_work_days_month | Integer | No | Default: 26 | Used for hourly rate derivation (BR-009) |
| standard_hours_day | Integer | No | Default: 8 | Used for hourly rate derivation (BR-009) |
| variance_alert_threshold | Decimal | No | Default: 0.20 (20%) | Per-group configurable (FR-34) |
| night_window_start | Time | No | Default: 22:00 | Night shift window for BR-007 |
| night_window_end | Time | No | Default: 06:00 | Night shift window for BR-007 |

### Search and Filter

- Filter by: legal_entity_id (default: current context), status (ACTIVE / INACTIVE / ALL)
- Sort by: code (asc/desc), name (asc/desc), created_at (asc/desc)
- Search: free-text on code or name

### List View Columns

| Column | Notes |
|--------|-------|
| Code | Link to detail page |
| Name | |
| Pay Calendar | Calendar name |
| Currency | |
| Worker Count | Active enrolled workers |
| Status | Badge: ACTIVE (green) / INACTIVE (grey) |
| Actions | Edit, Deactivate (if ACTIVE and no running periods) |

### Validation Rules

- **Duplicate code**: "PayGroup code must be unique within this legal entity."
- **Missing pay calendar**: "Pay calendar is required. Create one in Pay Calendars."
- **Deactivation blocked**: "PayGroup cannot be deactivated: it has an open or locked pay period. Close or lock all periods first."
- **Calendar mismatch**: "Selected pay calendar does not belong to this legal entity."

## Ambiguity / Open Questions

- AQ: Can a pay group be assigned multiple currencies (e.g., USD + VND for expat workers)? Decision assumed: single currency per pay group; multi-currency is out of scope V1.
- AQ: Who can deactivate a pay group — PAYROLL_ADMIN only or also HR_MANAGER? Decision assumed: PAYROLL_ADMIN only.
