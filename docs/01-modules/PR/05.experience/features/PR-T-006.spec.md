# PR-T-006: Bank Payment File

**Type**: T (Transaction)
**Priority**: P0
**BC**: BC-04 — Payroll Output and Disbursement
**Actors**: PAYROLL_ADMIN (primary), FINANCE_MANAGER (download and send)
**Linked Stories**: US-030
**API Endpoints**:
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/bank-files` — Initiate file generation
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/bank-files/{file_id}` — Get file status
- `GET /api/v1/payroll-usecase/payroll-runs/{run_id}/bank-files/{file_id}/download` — Download file
- `POST /api/v1/payroll-usecase/payroll-runs/{run_id}/bank-files/{file_id}/mark-sent` — Mark as sent

---

## Overview

After a payroll run is APPROVED (or LOCKED), the Payroll Admin generates a bank payment file for net salary disbursement. The file is formatted per the target bank's specification. Multiple banks may be active for one legal entity; the admin can generate per-bank or generate all at once.

Bank file generation is a V1 manual workflow. V2 will integrate directly with bank APIs.

---

## Pre-Conditions

Bank payment file generation requires:
- Payroll run status = APPROVED or LOCKED
- At least one worker in the run has a non-zero net pay

---

## Workflow Steps

### Step 1: Bank Selection

Screen shows available banks registered for this legal entity:

| Bank | Workers | Total Net |
|------|---------|-----------|
| Vietcombank (VCB) | 834 | 4,231,560,000 VND |
| BIDV | 201 | 1,022,340,000 VND |
| Techcombank (TCB) | 165 | 882,100,000 VND |
| No bank on record | 0 | — |

Selection options:
- Radio: select one specific bank
- [Generate All] button: generates one file per bank in one action

Workers with no bank account on record: shown as a warning row at the bottom. These workers are excluded from all files. Admin must update bank accounts in CO before proceeding, or proceed and handle these workers manually.

---

### Step 2: Preview

After selecting a bank, admin sees the payment preview table:

| Column | Notes |
|--------|-------|
| No. | Row number |
| Worker Name | |
| Employee ID | |
| Bank Account | Masked (show last 4 digits): ****1234 |
| Bank Name | Full bank name for the account |
| Net Amount (VND) | |

Summary footer: Total workers: [N] | Total amount: [amount] VND

Searchable: free-text on worker name or employee ID.

[View Full Account] link per row: shows unmasked account number in a confirmation modal (audit-logged access).

Export preview: [Download Preview CSV] — for admin pre-check before generating the official file.

---

### Step 3: Generate File

[Generate File] button: initiates async file generation.

Progress: spinner + "Generating [BankName] payment file..." 

On completion: file record appears in the Bank Files list with status READY_TO_DOWNLOAD.

**File generation is idempotent for the same run + bank combination.** If a file has already been generated for this bank, a warning shows: "A payment file for [BankName] was already generated on [date]. Generate a new file only if the previous one is void." [Generate New File] button requires explicit confirmation.

---

### Step 4: Download

[Download] button on each READY_TO_DOWNLOAD file record.

**File formats per bank**:

| Bank | Format | Notes |
|------|--------|-------|
| VCB (Vietcombank) | CSV (VCB iTransfer spec) | Columns per VCB corporate banking specification |
| BIDV | XML (BIDV DirectConnect spec) | XML schema per BIDV corporate spec |
| TCB (Techcombank) | CSV (TCB Payroll Upload spec) | Columns per TCB B2B spec |

**File header content** (common across formats):
- Legal entity name and tax code
- Pay group name
- Period (month/year)
- Total workers
- Total net amount
- Generated at timestamp
- Generated by (admin name)

**Per-worker row content**:
- Sequential row number
- working_relationship_id
- Worker full name
- Bank account number (full, unmasked in the file)
- Bank branch code (if required by bank spec)
- Net amount
- Currency: VND
- Transaction reference: `{legal_entity_code}/{period_yyyymm}/{run_id}/{worker_id}`
- Memo / narration: "Luong [month] [year] - [employee_id]"

---

### Step 5: Mark as Sent

After the admin has uploaded the file to the bank's portal, they mark it sent in xTalent:

[Mark as Sent] button on each READY_TO_DOWNLOAD or DOWNLOADED file.

Confirmation modal:
- "Confirm that the [BankName] payment file has been uploaded to the bank and is being processed."
- Date sent field: defaults to today; editable.
- Reference number field: optional bank transaction reference for tracking.

On confirmation: file status → SENT. This is recorded in the audit log.

## Error Handling

| Scenario | Behavior |
|----------|---------|
| Worker missing bank account | Worker excluded from all files; listed in warning panel "N workers have no bank account and will not be included." |
| File generation fails | Error shown on file record; [Retry] button available |
| Admin attempts generation before run is APPROVED | Button disabled with message: "Bank payment files can only be generated after the run is approved." |
| Duplicate file generation | Warning modal before proceeding; previous file retained in history |
| Downloaded file opened in wrong encoding | File encoding note in download modal: "This file uses UTF-8 encoding. Open with Excel: Data > From Text/CSV, select UTF-8." |
