# PR-A-004: Integrity Verification Report

**Type**: A — Analytics
**Priority**: P0
**BC**: BC-07 Audit Trail
**Actors**: PLATFORM_ADMIN
**Linked Stories**: US-047
**API Endpoints**: `GET /audit/integrity-verification`, `POST /audit/integrity-verification/trigger`

---

## Overview

Displays SHA-256 hash verification results for locked payroll periods. Any hash mismatch indicates potential data tampering and must be treated as a P0 security incident. Platform Admin can trigger manual re-verification at any time.

---

## Data Sources

| Source Table | Usage |
|-------------|-------|
| `integrity_hash_record` | hash_value, verified_at, verification_status (MATCH/MISMATCH/PENDING) |
| `payroll_result` | Current record hash recomputed at verification time |
| `pay_period` | period status (only LOCKED periods have hashes) |

---

## Metrics

| Metric | Definition |
|--------|-----------|
| Total Locked Periods | COUNT of pay_period where status = LOCKED |
| Verified Count | COUNT of integrity_hash_record where status = MATCH |
| Mismatch Count | COUNT where status = MISMATCH — triggers ALERT |
| Pending Count | COUNT where status = PENDING (not yet verified) |
| Last Verification Run | MAX(verified_at) |

---

## Verification Logic

At period lock, `integrity_hash_record` is created for each payroll_result with the SHA-256 hash of the serialised result. During verification (nightly job or manual trigger):
1. Re-serialise each payroll_result in the same canonical format
2. Compute SHA-256
3. Compare with stored `hash_value`
4. Update `verification_status` to MATCH or MISMATCH

---

## Alert Banner

**If ANY mismatch exists**: Red alert banner at top of page:
> "⚠️ SECURITY ALERT: [N] payroll records have failed integrity verification for period [MONTH YEAR] in [LEGAL ENTITY]. Immediately investigate and contact your system administrator."

The banner persists until all mismatches are acknowledged and investigated (acknowledgement requires PLATFORM_ADMIN role + written reason).

---

## Manual Verification Trigger

"Verify Now" button (per period row): triggers async re-verification job for all records in that period. Progress shown as spinner → refreshes status counts on completion.

"Verify All Pending" button: triggers re-verification for all PENDING records across all locked periods.

## Business Rules Applied

| Rule | Description |
|------|-------------|
| BR-096 | SHA-256 hash generated at PayPeriod LOCK |
