How This Event Works

1
Event triggers on schedule (last day of month at 11:00 PM)
2
Finds eligible classes based on scope filters (VN, ACCOUNT mode, ACTIVE)
3
Reads policy rules from each Leave Class's linked Policy (accrualRuleJson)
4
Processes each account (LeaveInstant) under eligible classes
5
Creates movements with quantities from policy rules

💡 Policy Resolution
When this event runs, it reads the accrualRuleJson from each Leave Class's linked Policy. Different classes can have different accrual rates (e.g., 1.75 vs 2.0 vs 2.5 days/month) based on their policies.
Execution Conditions - Event will only run if ALL conditions are met. Leave empty to run unconditionally.
