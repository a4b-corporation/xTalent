Architecture

The Four-Layer Model

Layer 1

Ontology-Driven Knowledge Layer

The "semantic brain" — enabling AI Agents to understand HR domain as deeply as a subject matter expert.

Domain Ontology

Formal HR entity models in LinkML/OWL

Semantic Layer

Business terms → technical data mapping

Knowledge Graph

Employee → Position → Dept relationships

Rules Engine

Encoded labor law & policy guardrails

Cognitive API

"SME-as-a-Service" for any Agent

🔑 Without this layer, AI is surface-level automation. With it, AI becomes a domain-aware decision partner.

Layer 2

Adaptive UI Layer — The 30-60-10 Model

The right interface, to the right user, at the right time — balancing consistency with flexibility.

Core UI (Fixed) — 30%

Navigation, design system, brand identity. Built by developers for maximum quality.

Configurable UI — 60%

JSON/Template-driven views. Admins customize without code deployment.

Generative UI (A2UI) — 10%

AI Agents create interfaces on-demand for unpredictable long-tail needs.

Powered by: A2UI (Google) + AG-UI (Event Protocol) + MCP (Data Access)

Layer 3

Intelligent Backend Layer

From APIs to agent teams — processing business logic through traditional and AI-powered orchestration.

Headless API

REST/GraphQL stateless CRUD — serves all clients

Workflow Engine

Visual workflow designer for HR processes — zero-code customization

Agent Orchestration

Multi-agent teams: Recruit, Comply, Payroll, Analytics

Policy & Governance

Agent RBAC, audit logs, kill switches, HITL gates

⚡ Workflow Engine Impact: Customer requests "3-level approval for leave > 5 days" → Traditional: 2-week release. xTalent: Configure & deploy instantly.

Layer 4

Multi-Modal Data Layer

The right storage paradigm for every type of data — from strict transactional records to AI-native stores.

Core RDBMS

PostgreSQL — ACID, strict schema, audit trail

Flex Store

SurrealDB — schemaless custom fields, dynamic forms

Graph Store

Neo4j — org charts, skill maps, GraphRAG

Metadata & Audit

Catalog + lineage — AI decision traceability

AI-Native Store

Vector DB — embeddings, semantic search, RAG

🔒 Metadata & Audit is mandatory for HR compliance — labor law, data privacy, GDPR/PDPA, AI transparency.

Roadmap

Three Phases to Evolution

Phase 1: Foundation

Months 1-6
  • Core HR Domain Ontology (LinkML)
  • Knowledge Graph — org structure
  • Headless REST/GraphQL APIs
  • Core UI — design system
  • RDBMS + Flex Store + Metadata

Phase 2: Intelligence

Months 7-12
  • Workflow Engine — visual designer
  • Agent Team: Onboard, Comply, Payroll
  • Configurable UI (JSON-driven)
  • Full Knowledge Graph + GraphRAG
  • Vector DB — semantic search

Phase 3: Evolution

Months 13-18
  • Generative UI — A2UI integration
  • Multi-Agent cross-functional teams
  • MAPE-K self-optimization loops
  • Predictive HR Analytics
  • Auto-evolving rules & workflows
