# 1.7 — ConfigScope: Nhóm phạm vi cấu hình đa quốc gia

**Schema:** `comp_core`  
**Tables:** `comp_core.config_scope` · `comp_core.config_scope_member`  
**Tham chiếu DBML:** [`4.TotalReward.V5.dbml`](./4.TotalReward.V5.dbml) — Section 0. Configuration Scope  
**Last Updated:** 2026-04-06

---

## Mục đích nghiệp vụ

`config_scope` là **cơ chế scoping nâng cao (Phase 2)** cho phép nhóm các quốc gia, Legal Entity, Business Unit thành các *scope có tên* để tái sử dụng trong config của các master data entity.

Thay vì mỗi entity (salary_basis, pay_component_def, comp_plan...) khai báo riêng `country_code + legal_entity_id`, Phase 2 dùng một `config_scope_id` duy nhất và tất cả entities trong scope đó cùng chia sẻ một definition.

---

## Khi nào dùng Config Scope?

| Phase | Deployment | Cơ chế | Dùng khi |
|-------|-----------|--------|---------|
| **Phase 1** | Light Touch | `country_code` + `legal_entity_id` trực tiếp trên entity | 1–3 quốc gia, simple setup |
| **Phase 2** | Config Scope | `config_scope_id` trên entity | 5+ quốc gia, scope hierarchy cần inheritance |

**Resolution Priority (khi cả hai tồn tại):**
```
config_scope_id IS NOT NULL  →  ưu tiên (Phase 2 win)
config_scope_id IS NULL      →  fallback về country_code + legal_entity_id (Phase 1)
config_scope_id IS NULL AND country_code IS NULL  →  global (tất cả)
```

## Schema — `config_scope`

| Field | Type | Ý nghĩa |
|-------|------|---------|
| `id` | uuid | Primary key |
| `scope_code` | varchar(50) | Mã duy nhất. Ví dụ: `VN_DEFAULT`, `SG_DEFAULT`, `APAC_REGION`, `VN_TECH_BU` |
| `scope_name` | varchar(200) | Tên hiển thị. Ví dụ: "Vietnam Default Scope" |
| `scope_type` | varchar(20) | `COUNTRY` \| `LEGAL_ENTITY` \| `BUSINESS_UNIT` \| `HYBRID` |
| `country_code` | char(2) | ISO 3166-1. Dùng cho `scope_type = COUNTRY / LEGAL_ENTITY`. `NULL` = multi-country scope |
| `legal_entity_id` | uuid | Dùng cho `scope_type = LEGAL_ENTITY`. `NULL` = tất cả LE trong country |
| `business_unit_id` | uuid | Dùng cho `scope_type = BUSINESS_UNIT`. |
| `priority` | int | Mức ưu tiên khi resolve — cao hơn = cụ thể hơn. Xem convention bên dưới |
| `parent_scope_id` | uuid | FK tự tham chiếu — xây dựng cây inheritance |
| `inherit_flag` | boolean | `true` = kế thừa config từ parent scope nếu chưa set |
| `description` | text | Mô tả |
| `is_active` | boolean | `false` = scope đã retire |

### `scope_type` Values

| type | Ý nghĩa | country_code | legal_entity_id | business_unit_id |
|------|---------|:---:|:---:|:---:|
| `COUNTRY` | Toàn quốc gia | ✅ required | null | null |
| `LEGAL_ENTITY` | Cụ thể một LE | ✅ required | ✅ required | null |
| `BUSINESS_UNIT` | Cụ thể một BU | ✅ | ✅ | ✅ required |
| `HYBRID` | Nhiều quốc gia/LE | null | null | null (dùng config_scope_member) |

### Priority Convention

| priority | Scope Level | Ví dụ scope_code |
|----------|------------|-----------------|
| 0 | GLOBAL | `GLOBAL_DEFAULT` |
| 10 | COUNTRY | `VN_DEFAULT`, `SG_DEFAULT` |
| 20 | LEGAL_ENTITY | `VN_ENTITY_A`, `VN_ENTITY_B` |
| 30 | BUSINESS_UNIT | `VN_TECH_BU`, `VN_OPS_BU` |

**Resolution:** Most specific (highest priority) scope wins.

---

## Schema — `config_scope_member`

Bridge table cho **HYBRID scope** — một scope có thể gom nhiều quốc gia/LE thành một nhóm.

| Field | Type | Ý nghĩa |
|-------|------|---------|
| `id` | uuid | Primary key |
| `scope_id` | uuid | FK → `comp_core.config_scope` |
| `member_type` | varchar(20) | `COUNTRY` \| `LEGAL_ENTITY` \| `BUSINESS_UNIT` |
| `country_code` | char(2) | Khi `member_type = COUNTRY` |
| `legal_entity_id` | uuid | Khi `member_type = LEGAL_ENTITY` |
| `business_unit_id` | uuid | Khi `member_type = BUSINESS_UNIT` |

**Unique constraint:** `(scope_id, member_type, country_code, legal_entity_id)` — không duplicate members.

## Scope Inheritance (parent_scope_id)

```
config_scope hierarchy với inherit_flag=true:

GLOBAL (priority=0, parent=null)
  └── VN_DEFAULT (priority=10, parent=GLOBAL, country=VN)
        └── VN_ENTITY_A (priority=20, parent=VN_DEFAULT, le=uuid-a)
              └── VN_TECH_BU (priority=30, parent=VN_ENTITY_A, bu=uuid-tech)

Resolution cho entity trong VN_TECH_BU:
  1. Check VN_TECH_BU scope → nếu config đã set → dùng
  2. inherit_flag=true → check VN_ENTITY_A → config inherit
  3. inherit_flag=true → check VN_DEFAULT → config inherit
  4. inherit_flag=true → check GLOBAL → config inherit
  5. Không có gì → dùng system default
```

**Khi nào inherit_flag=false:** Khi một LE có policy hoàn toàn khác biệt, không muốn kế thừa từ parent. Ví dụ: VN_SPECIAL_ENTITY không inherit từ VN_DEFAULT vì có chế độ lương riêng hoàn toàn.

## Ví dụ Setup Thực tế

### Trường hợp 1: Single Country (Phase 1 — không cần config_scope)

```
salary_basis:
  code=MONTHLY_VND_FULL_TIME
  country_code=VN          ← inline field trực tiếp
  legal_entity_id=null     ← all VN LEs
  config_scope_id=null     ← Phase 1, không dùng scope
```

### Trường hợp 2: Multi-Country (Phase 2 — dùng config_scope)

```
config_scope:
  GLOBAL (priority=0)
    └─ VN_DEFAULT (priority=10, country=VN)
    └─ SG_DEFAULT (priority=10, country=SG)
    └─ TH_DEFAULT (priority=10, country=TH)

salary_basis:
  code=MONTHLY_APAC_UNIFIED
  config_scope_id → GLOBAL  ← áp dụng cho tất cả APAC countries

comp_plan:
  code=MERIT_2025_VN
  config_scope_id → VN_DEFAULT  ← chỉ cho VN
  
comp_plan:
  code=MERIT_2025_SG
  config_scope_id → SG_DEFAULT  ← chỉ cho SG
```

### Trường hợp 3: HYBRID Scope — gom nhiều LE vào 1 scope

```
config_scope:
  code=APAC_REGION (type=HYBRID, priority=10)

config_scope_member:
  (APAC_REGION, COUNTRY, VN)
  (APAC_REGION, COUNTRY, SG)
  (APAC_REGION, COUNTRY, TH)
  (APAC_REGION, COUNTRY, MY)

comp_plan:
  code=EQUITY_CORRECTION_APAC
  config_scope_id → APAC_REGION  ← áp dụng cho cả 4 quốc gia APAC
```

## Entities Sử Dụng config_scope_id

Tất cả definition-level entities trong TR domain đều hỗ trợ `config_scope_id`:

| Entity | Qua config_scope_id | Phase 1 inline fallback |
|--------|---------------------|------------------------|
| `salary_basis` | ✅ | `country_code + legal_entity_id` |
| `pay_component_def` | ✅ | `country_code` (no LE — component là country-level) |
| `comp_plan` | ✅ | `country_code + legal_entity_id` |
| `calculation_rule_def` | ✅ | `country_code + jurisdiction` |
| `grade_ladder` | N/A | Ladders are typically global |
| `pay_range` | N/A | Pay range scope dùng `scope_type+scope_uuid` riêng |

---

## Business Rules

### Rule 1: config_scope_id beats inline fields

Khi entity có cả `config_scope_id` lẫn `country_code`, `config_scope_id` win. Application phải enforce đừng set cả hai cùng lúc để tránh confusion.

### Rule 2: Scope Code là Immutable sau khi Referenced

Khi một entity (salary_basis, comp_plan...) đã reference `config_scope.id`, scope_code không nên thay đổi (mặc dù DBML không enforce). Thay vào đó tạo scope mới nếu cần.

### Rule 3: HYBRID scope cần config_scope_member

Scope `type=HYBRID` với `country_code=NULL` → members phải được khai báo đầy đủ trong `config_scope_member`. Application phải validate.

## Common Gotchas

| Gotcha | Cách tránh |
|--------|-----------|
| Set cả `config_scope_id` lẫn `country_code` | Application phải warn/block — chỉ dùng 1 approach |
| HYBRID scope thiếu member records | Validate: `type=HYBRID AND country_code IS NULL` → phải có ≥1 member |
| Circular parent reference | Validate: parent chain không vòng tròn trước khi save |
| Phase 1 entities bị bỏ qua khi dùng scope | Resolution logic phải kiểm tra `config_scope_id` TRƯỚC rồi mới fallback về inline |
