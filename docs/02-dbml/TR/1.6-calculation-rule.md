**Tham chiếu DBML:** [`4.TotalReward.V5.dbml`](./4.TotalReward.V5.dbml) — section `9. Calculation Rules`  
**Last Updated:** 2026-04-06

## Mục đích nghiệp vụ

`calculation_rule_def` là kho chứa **các quy tắc tính toán thuộc HR-policy layer** (TR domain). Các rules áp dụng khi tính lương projected/Gross trước khi chuyển sang Payroll (PR) để tính Net.

**Domain Boundary — Critical:**

```
TR domain (file này):              PR domain (pay_master.statutory_rule):
  PRORATION                          TAX (PIT brackets)
  ROUNDING                           SOCIAL_INSURANCE (BHXH, BHYT, BHTN)
  FOREX                              OVERTIME (OT multipliers per Labor Code)
  ANNUALIZATION                      GROSS_TO_NET (full net pay pipeline)
  COMPENSATION_POLICY
```

## Schema — `calculation_rule_def`

| Field | Type | Ý nghĩa |
|-------|------|---------|
| `id` | uuid | Primary key (version-specific) |
| `code` | varchar(50) | Mã bất biến qua versions. Ví dụ: `VN_PRORATION_WORKDAYS` |
| `name` | varchar(200) | Tên mô tả |
| `rule_category` | varchar(30) | HR-policy category — xem taxonomy |
| `rule_type` | varchar(30) | `FORMULA` \| `LOOKUP_TABLE` \| `CONDITIONAL` \| `RATE_TABLE` \| `PROGRESSIVE` |
| `country_code` | char(2) | `NULL` = global |
| `formula_json` | jsonb | Actual calculation logic |
| `legal_reference` | text | Tham chiếu pháp luật nếu có |
| `version_number` | int | SCD Type 2 version |
| `previous_version_id` | uuid | FK về version trước |
| `is_current_version` | boolean | `true` = current |
| `effective_start` | date | Ngày có hiệu lực |
| `effective_end` | date \| null | `NULL` = active |

### `rule_category` Taxonomy (TR Domain Only)

| category | Mô tả | Ví dụ |
|----------|-------|-------|
| `PRORATION` | Tính phần lương khi nghỉ/gia nhập giữa kỳ | `amount × (working_days / standard_days)` |
| `ROUNDING` | Làm tròn số tiền | Làm tròn đến 1,000 VND |
| `FOREX` | Chuyển đổi ngoại tệ | USD → VND theo tỷ giá daily |
| `ANNUALIZATION` | Convert frequency Monthly ↔ Annual | Monthly × 12 = Annual |
| `COMPENSATION_POLICY` | Policy nội bộ | 13th month = tháng 12 × tenure/12 |

### `formula_json` Examples

```json
// PRORATION
{"expression": "amount * (actual_workdays / standard_workdays_per_month)",
 "params": {"standard_workdays_per_month": {"source": "country_config"}}}

// ROUNDING — làm tròn đến 1000 VND
{"method": "ROUND_NEAREST", "unit": 1000, "currency": "VND"}

// ANNUALIZATION
{"expression": "amount * 12"}

// COMPENSATION_POLICY — 13th month
{"type": "CONDITIONAL",
 "conditions": [
   {"if": "tenure_months >= 12", "then": {"formula": "monthly_base_salary"}},
   {"if": "tenure_months < 12",  "then": {"formula": "monthly_base_salary * tenure_months / 12"}}
 ]}
```

## Calculation Flow (TR Scope)

```
Nhân viên vào giữa tháng 10/07/2025, lương tháng 20M + phụ cấp 730K:

TR Layer:
  execution_order=1: PRORATION
    LUONG_CO_BAN: 20,000,000 × (17/26) = 13,076,923
    PHU_CAP_AN_TRUA: override_params={force_prorate:false} → 730,000 (không prorate)
  
  execution_order=2: ROUNDING
    LUONG_CO_BAN: ROUND(13,076,923 / 1000) × 1000 = 13,077,000

  TR Gross Total: 13,807,000 VND → gửi sang PR domain

PR Layer:
  BHXH (10.5%) + BHYT (1.5%) + BHTN (1%) + PIT → Net Pay
```

---

## Business Rules

### Rule 1: TR chỉ tính Gross — không tính Net
TR engine output = Gross amount → PR engine owns Gross→Net pipeline.

### Rule 2: execution_order quyết định sequence
PRORATION trước → ROUNDING sau. Không thể đảo ngược.

### Rule 3: override_params ở bridge table
Component/basis có thể override params: `{"force_prorate": false}` cho phụ cấp ăn trưa không prorate.

### Rule 4: SCD Type 2 — không sửa current version
Khi policy thay đổi → tạo version mới, set `effective_end` cho version cũ.
