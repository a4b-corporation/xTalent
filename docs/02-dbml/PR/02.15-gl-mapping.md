# gl_mapping — Ánh xạ GL Account (GL Mapping)

> **Schema:** `pay_master.gl_mapping`
> **DDD Classification:** Value Object
> **SCD-2:** `effective_start_date / effective_end_date / is_current_flag`
> **Changed:** JUL 2025 (initial)

---

## 1. Config những gì?

`gl_mapping` ánh xạ từng `pay_element` (hoặc tổ hợp element × country/LE) sang tài khoản kế toán GL cụ thể. Cho phép cùng 1 element nhưng mapping khác nhau tùy LE hay country.

### Fields

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `element_id` | uuid FK NOT NULL | Element được map | FK → `pay_master.pay_element` |
| `legal_entity_id` | uuid FK | LE cụ thể. `NULL` = global mapping | FK → `org_legal.entity` |
| `country_code` | char(2) | Country cụ thể. `NULL` = global | `VN`, `SG` |
| `gl_account_code` | varchar(50) NOT NULL | Tài khoản GL | `622001`, `334`, `3383` |
| `gl_account_name` | varchar(100) | Tên tài khoản | `Phải trả CNV`, `BHXH phải nộp` |
| `debit_credit` | char(1) NOT NULL | Loại bút toán | `D` (Debit) hoặc `C` (Credit) |
| `cost_center_code` | varchar(50) | CC mặc định cho element này | `CC-PRODUCTION` |
| `description` | text | Mô tả mapping | |
| `effective_start_date` | date | | |
| `effective_end_date` | date | `NULL` = đang active | |
| `is_current_flag` | boolean | SCD-2 | |
| `is_active` | boolean | default `true` | SCD-2 active record |


---

## 2. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-GL01** | Priority: LE-specific > country-specific > global. Engine dùng mapping cụ thể nhất có thể tìm được. |
| **BR-PR-GL02** | Mỗi element thường có 2 records: 1 Debit (chi phí) + 1 Credit (phải trả). Ví dụ: BASIC_SALARY → Debit `622001` (chi phí tiền lương) + Credit `334` (phải trả NLĐ). |
| **BR-PR-GL03** | Khi kế hoạch kế toán thay đổi → tạo SCD-2 record mới. Records cũ giữ nguyên để đảm bảo journal entries lịch sử vẫn đúng mapping. |

## 4. Design Notes

> [!NOTE]
> **GL mapping ≠ journal posting.** `gl_mapping` chỉ config mapping rules. Actual journal entries được tạo bởi accounting module sau khi payroll confirm, dựa trên `pay_engine.costing` + `gl_mapping`.

> [!NOTE]
> **Debit/Credit logic:** EARNING elements → D=Chi phí (6xx), C=Phải trả (334). DEDUCTION elements → D=Phải trả (334), C=Phải nộp (3383, 3335). Chi phí chủ SDLĐ → D=Chi phí (6xx), C=Phải nộp (3383).
