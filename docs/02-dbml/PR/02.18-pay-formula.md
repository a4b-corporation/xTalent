# pay_formula — Công thức Tính lương (Pay Formula)

> **Schema:** `pay_master.pay_formula`
> **DDD Classification:** Value Object
> **Changed:** JUL 2025 (initial) | 07Apr2026 (Change 39: `script text` → `script_json jsonb` + `scope` field) | 13Apr2026 (Change 47: align field names — `version_no`, `name varchar(255)`)

---

## 1. Config những gì?

`pay_formula` lưu trữ các **công thức tính lương** dùng trong engine. Sau Change 39, công thức được lưu dưới dạng `script_json jsonb` thay vì text thuần — cho phép lưu metadata ngôn ngữ, version, và params cùng với script.

> **Engine decision (07Apr2026):**
> - **MVEL** → formula layer: expression đơn giản (proration, allowance calc)
> - **Groovy** → engine layer: logic phức tạp (PIT progressive, SI cap, pipeline)
> - Không có DB enum constraint — application layer route đến executor phù hợp dựa trên `script_json.lang`

### Fields

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `code` | varchar(50) UNIQUE | Mã công thức | `FML_MONTHLY_PRORATE`, `FML_PIT_PROGRESSIVE` |
| `name` | varchar(255) | Tên | `Proration theo ngày làm việc` |
| `scope` | varchar(20) | Gợi ý scope dùng. Không ràng buộc | `ELEMENT`, `PROFILE`, `STATUTORY`, `ANY` |
| `script_json` | jsonb NOT NULL | Công thức + metadata | Xem cấu trúc bên dưới |
| `description` | text | Mô tả | |
| `is_active` | boolean | | `true` |
| `version_no` | int | Phiên bản formula | `1`, `2` — *changed 13Apr2026: `version smallint` → `version_no int`* |
| `created_at` | timestamptz | | |
| `updated_at` | timestamptz | | |


### Cấu trúc `script_json`

```jsonc
{
  "lang": "MVEL",              // MVEL | GROOVY | JSLT | DROOLS_DRL (future)
  "content": "...",             // Script/expression text
  "engine_version": "mvel-2.4.2", // Optional: pin engine version
  "params": {                   // Optional: input params documentation
    "monthly_salary": "decimal — from compensation.basis",
    "actual_work_days": "int — from ta.timesheet",
    "standard_work_days": "int — from pay_calendar (26)"
  },
  "return_type": "decimal",     // Optional: expected return type
  "test_cases": [               // Optional: unit test cases for validation
    {"input": {"monthly_salary": 10000000, "actual": 20, "standard": 26}, "expected": 7692307.69}
  ]
}
```

## 2. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-PF01** | `script_json.lang` quyết định executor. MVEL cho formula đơn. Groovy cho pipeline phức tạp. Application không được hard-code executor — luôn đọc từ `script_json.lang`. |
| **BR-PR-PF02** | Backward compat migration: script text cũ → `{"lang":"MVEL","content":"<old_text>"}`. |
| **BR-PR-PF03** | Formula code được reference bởi `pay_element.formula_json → {"ref":"FML_CODE"}` hoặc `pay_profile_element.override_formula_json → {"ref":"FML_CODE"}`. |
| **BR-PR-PF04** | `scope` chỉ là UI hint để admin filter. Engine không validate scope — application gợi ý nhưng không enforce. |
| **BR-PR-PF05** | `version` field: khi sửa đổi formula logic, tạo version mới (update `version` + `updated_at`) thay vì tạo record mới, trừ khi cần giữ cả 2 versions active song song. |

### Ví dụ 1: MVEL — Proration theo ngày làm việc

```json
{
  "code": "FML_MONTHLY_PRORATE",
  "name": "Proration lương tháng theo ngày làm việc",
  "scope": "ELEMENT",

    "lang": "MVEL",
    "content": "monthly_salary * actual_work_days / standard_work_days",
```

## 4. Query Patterns

```sql
-- Tất cả active formulas theo scope
SELECT code, name, scope,
       script_json->>'lang' AS language,
       script_json->>'engine_version' AS engine_ver
FROM pay_master.pay_formula
WHERE is_active = TRUE
  AND (scope = :scope OR scope IS NULL)
ORDER BY scope, code;

-- Elements nào đang dùng formula này?
SELECT pe.code AS element_code, pe.name
FROM pay_master.pay_element pe
WHERE pe.formula_json->>'ref' = :formula_code
  AND pe.is_current_flag = TRUE;
```

## 5. Design Notes

> [!IMPORTANT]
> **Không hard-code executor trong app.** Luôn đọc `script_json.lang` để route. Khi công ty quyết định migrate từ MVEL sang Groovy → chỉ cần update `script_json.lang` trên các records, không cần deploy code mới.

> [!NOTE]
> **`test_cases` trong script_json** là optional nhưng highly recommended cho formula quan trọng (PIT, severance). CI/CD pipeline có thể auto-run test cases mỗi khi formula được update.
