# payslip_template — Mẫu Phiếu Lương (Payslip Template)

> **Schema:** `pay_master.payslip_template`
> **DDD Classification:** Value Object
> **SCD-2:** `effective_start_date / effective_end_date / is_current_flag`
> **Changed:** JUL 2025 (initial) | 13Apr2026 (Change 47: add `template_type`, `logo_url`, `header_text`, `footer_text`; fix `locale` → `locale_code`; note SCD-2)


---

## 1. Config những gì?

`payslip_template` định nghĩa **format và nội dung hiển thị** của phiếu lương. Mỗi `pay_group` hoặc `pay_profile` có thể dùng template riêng — nhà máy vs văn phòng thường có layout khác nhau.

### Fields

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `code` | varchar(50) UNIQUE | Mã template | `TPL_VN_STANDARD`, `TPL_FACTORY_DETAILED` |
| `name` | varchar(255) | Tên | `Phiếu lương chuẩn VN` |
| `template_type` | varchar(20) | Loại render engine | `PDF`, `HTML`, `XLSX` — *NEW 13Apr2026* |
| `locale_code` | varchar(10) | Ngôn ngữ + locale | `vi_VN`, `en_US`, `en_SG` — *changed 13Apr2026: `locale` → `locale_code`* |
| `template_json` | jsonb | Cấu hình layout và sections | Xem cấu trúc bên dưới |
| `logo_url` | varchar(255) | URL logo công ty | `https://cdn.company.com/logo.png` — *NEW 13Apr2026* |
| `header_text` | text | Tiêu đề custom | `PHIỪU LƯƠNG THÁNG {{month}}/{{year}}` — *NEW 13Apr2026* |
| `footer_text` | text | Chân trang | `Chữ ký nhân viên: ___________` — *NEW 13Apr2026* |
| `description` | text | Mô tả template | |
| `is_active` | boolean | | `true` |
| `is_default` | boolean | Template mặc định cho LE | `true` |
| `effective_start_date` | date | SCD-2 — ngày hiệu lực | |
| `effective_end_date` | date | SCD-2 — `NULL` = đang active | |
| `is_current_flag` | boolean | SCD-2 | |

## 2. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-PT01** | `is_default = true` chỉ được có 1 template per LE. Application enforces unique constraint. |
| **BR-PR-PT02** | `show_ytd_balances = true` → phiếu lương hiển thị YTD PIT và YTD gross — thường yêu cầu bởi employees để khai thuế. |
| **BR-PR-PT03** | Template type `PDF` dùng rendering engine (Jasper, iText...). `HTML` dùng email delivery. `XLSX` dùng cho bulk export. |

## 3. Ví dụ — 2 templates điển hình

**Template văn phòng (đơn giản):**
```json
{
  "code": "TPL_VN_OFFICE_SIMPLE",
  "locale": "vi_VN",
  "template_type": "PDF",
  "sections": [
    {"title": "Thu nhập", "show_elements": ["BASIC_SALARY", "ALL_ALLOWANCES"]},
    {"title": "Khấu trừ", "show_elements": ["BHXH_EE_VN", "BHYT_EE_VN", "BHTN_EE_VN", "PIT_WITHHOLD_VN"]},
    {"title": "Thực lĩnh", "highlight": true}
  ],
  "show_ytd_balances": true
}
```

**Template nhà máy (chi tiết OT + sản phẩm):**
```json
{
  "code": "TPL_VN_FACTORY_DETAIL",
  "locale": "vi_VN",
  "template_type": "PDF",
  "sections": [
    {"title": "Thông tin ca làm việc", "fields": ["shift_pattern", "actual_workdays", "ot_hours"]},
    {"title": "Lương sản phẩm", "show_elements": ["PIECE_RATE_PAY", "PIECE_RATE_SUPPLEMENT"]},
    {"title": "OT Pay", "show_elements": ["OT_WEEKDAY_PAY", "OT_WEEKEND_PAY", "OT_HOLIDAY_PAY"]},
    {"title": "Phụ cấp", "show_elements": ["MEAL_ALLOWANCE", "HAZARD_ALLOWANCE", "NIGHT_PREMIUM"]},
    {"title": "Khấu trừ", "element_classifications": ["DEDUCTION", "TAX"]},
    {"title": "Thực lĩnh", "highlight": true}
  ],
  "show_ytd_balances": false
}
```

## 4. Design Notes

> [!NOTE]
> **Template là config, không phải engine logic.** Rendering engine đọc `template_json` và `pay_engine.result` để render phiếu lương. `payslip_template` không có code xử lý — chỉ là configuration data.
