# balance_def — Định nghĩa Balance (Balance Definition)

> **Schema:** `pay_master.balance_def`
> **DDD Classification:** Value Object
> **Changed:** JUL 2025 (initial)

---

## 1. Config những gì?

`balance_def` định nghĩa các loại **số dư tích lũy** mà payroll engine theo dõi cho mỗi element: run-to-date, quarter-to-date, year-to-date, life-to-date. Được dùng để tính cumulative balance (YTD tax, annual SI cap tracking...).

### Fields

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `code` | varchar(50) UNIQUE | Mã balance | `YTD_GROSS`, `YTD_PIT`, `QTD_OT_HOURS` |
| `name` | varchar(100) | Tên hiển thị | `Thu nhập gộp YTD`, `Thuế TNCN YTD` |
| `balance_type` | varchar(10) NOT NULL | Loại tích lũy | Xem enum bên dưới |
| `element_id` | uuid FK | Element được track (null = tổng hợp nhiều elements) | FK → `pay_master.pay_element` |
| `reset_freq_code` | varchar(20) | Khi nào reset về 0 | FK → `pay_master.pay_frequency` |
| `description` | text | Mô tả | |
| `is_active` | boolean | | `true` |

---

## 2. Enum — `balance_type`

| Giá trị | Tích lũy | Reset khi nào | Dùng cho |
|---------|---------|--------------|----------|
| `RUN` | Trong 1 payroll run | Mỗi run | Debug, retro check |
| `QTD` | Trong quý hiện tại | Đầu mỗi quý | Quarterly bonus tính lũy tiến |
| `YTD` | Trong năm | Đầu năm (01/01) | PIT quyết toán, SI annual cap (36tr/năm) |
| `LTD` | Life-to-date (toàn thời gian) | Không reset | Công nhân tích lũy sản lượng, seniority |

---

## 3. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-BD01** | `YTD_PIT` balance bắt buộc phải có và link `element_id → PIT_WITHHOLD`. Payroll engine đọc YTD PIT để quyết toán cuối năm. |
| **BR-PR-BD02** | `YTD_SI_BASE` balance track lương đóng BHXH tích lũy — khi reach trần (20 × lương cơ sở × 12 tháng) → engine tự dừng khấu trừ BHXH cho phần vượt. |
| **BR-PR-BD03** | `reset_freq_code = ANNUAL` → balance reset ngày 01/01 mỗi năm. `pay_engine.cumulative_balance` lưu giá trị thực tế. |

## 4. Ví dụ — Seed data chuẩn cho VN payroll

| code | balance_type | element | reset_freq | Mục đích |
|------|:-----------:|---------|:----------:|---------|
| `YTD_GROSS_EARN` | `YTD` | `BASIC_SALARY` (multi) | `ANNUAL` | Tổng thu nhập gộp năm (PIT base) |
| `YTD_PIT_WITHHELD` | `YTD` | `PIT_WITHHOLD_VN` | `ANNUAL` | Tổng thuế đã khấu trừ trong năm |
| `YTD_BHXH_BASE` | `YTD` | `BHXH_EE_VN` | `ANNUAL` | Lương đóng BHXH YTD (check vs trần 36tr) |
| `YTD_OT_HOURS` | `YTD` | `OT_WEEKDAY_PAY` | `ANNUAL` | Giờ OT tích lũy năm (check vs 300h cap) |
| `QTD_BONUS` | `QTD` | `BONUS` | `QUARTERLY` | Bonus QTD cho quarterly plan |
| `LTD_PIECE_UNITS` | `LTD` | null | null | Tổng sản phẩm LTD (seniority KPI) |

---

## 5. Design Notes

> [!NOTE]
> **`pay_engine.cumulative_balance`** là nơi lưu giá trị thực tế của balance này per (employee × period). `balance_def` chỉ là definition — không chứa values.

> [!NOTE]
> **YTD OT hours** track qua balance_def → khi đạt 300h (BLLĐ Điều 107) → engine emit warning (OT cap enforcement formal vẫn là TA responsibility, nhưng PR có thể second-check qua validation_rule).
