# pay_profile_rate_config — Cấu hình Đơn giá theo Chiều (Profile Rate Config)

> **Schema:** `pay_master.pay_profile_rate_config`
> **DDD Classification:** Value Object
> **SCD-2:** Không — versioned qua `effective_start_date / effective_end_date`
> **Changed:** 27Mar2026 (NEW — AQ-12 Hourly Rate Differentiation, Hybrid C+D) | 13Apr2026 (Change 47: align field names — `base_rate_amount`, `formula_id` FK, add `currency_code`)


---

## 1. Config những gì?

`pay_profile_rate_config` định nghĩa **đơn giá lương theo chiều (dimension)** ở mức profile. Đây là **Layer 2** trong kiến trúc 3-layer rate lookup cho hourly/piece-rate workers.

> **Kiến trúc 3-layer rate lookup:**
> ```
> Layer 1: worker_rate_override   ← per-worker exception (~5% workers)
> Layer 2: pay_profile_rate_config ← profile-level defaults [ĐÂY]
> Layer 3: statutory_rule          ← OT multipliers (law-driven)
> ```
> Engine resolve: Layer 1 → Layer 2 → Layer 3 (priority giảm dần).

### Nhóm 1 — Binding & Dimension

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `pay_profile_id` | uuid FK NOT NULL | Profile áp dụng | FK → `pay_master.pay_profile` |
| `rate_dimension` | varchar(30) NOT NULL | Chiều tính rate | Xem enum bên dưới |
| `rate_type` | varchar(20) NOT NULL | Cách xác định rate value | `FIXED`, `MULTIPLIER`, `FORMULA` |
| `is_active` | boolean | Có đang dùng không? | `true` |

### Nhóm 2 — Rate Values

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `base_rate_amount` | decimal(15,4) | Rate value. Ý nghĩa phụ thuộc `rate_type` | `50000.0000` (FIXED: VND/h) hoặc `1.5` (MULTIPLIER) — *changed 13Apr2026: `base_rate decimal(18,4)` → `base_rate_amount decimal(15,4)`* |
| `currency_code` | char(3) | Currency của rate. `NULL` = inherit từ `pay_profile` | `VND`, `USD` — *NEW 13Apr2026* |
| `statutory_rule_id` | uuid FK | Statutory rule cung cấp OT multiplier. Dùng khi rate_type=`MULTIPLIER` | FK → `pay_master.statutory_rule` |
| `formula_id` | uuid FK | FK đến `pay_formula` nếu rate_type=`FORMULA` | FK → `pay_master.pay_formula` — *changed 13Apr2026: `formula_code varchar(50)` (soft ref) → `formula_id uuid FK` (hard FK)* |


### Nhóm 3 — Thời gian

| Field | Type | Ý nghĩa |
|-------|------|---------|
| `effective_start_date` | date | Ngày hiệu lực |
| `effective_end_date` | date | Ngày hết hiệu lực. `NULL` = đang active |

## 2. Enum & Giá trị mặc định

### `rate_dimension` — Chiều tính rate

| Giá trị | Ý nghĩa | Ứng dụng |
|---------|---------|----------|
| `REGULAR` | Ca thường — baseline hourly rate | Tất cả hourly workers |
| `NIGHT` | Ca đêm (22:00–06:00) | Nhà máy 3 ca, bảo vệ, y tế |
| `OT_WEEKDAY` | OT ngày thường | Vượt quá giờ chuẩn ngày LV |
| `OT_WEEKEND` | OT cuối tuần | Thứ 7, Chủ nhật |
| `OT_HOLIDAY` | OT ngày lễ | BLLĐ × 3.0 |
| `HAZARDOUS` | Điều kiện làm việc độc hại/nguy hiểm | Khai mỏ, hóa chất, xây dựng |
| `SPECIALIZED` | Kỹ năng/bằng cấp đặc biệt | Thợ hàn chứng chỉ, lái xe hạng nặng |

### `rate_type`

| Giá trị | `base_rate` nghĩa là | Dùng khi |
|---------|----------------------|----------|
| `FIXED` | Số tiền tuyệt đối (VND/giờ) | Rate cố định không phụ thuộc lương cơ bản |
| `MULTIPLIER` | Hệ số nhân (× REGULAR rate) | OT = 1.5× base; Night = 1.3× base |
| `FORMULA` | Không dùng — xem `formula_code` | Rate phức tạp (theo khu vực, hiệu suất) |

## 3. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-PRC01** | Unique constraint: `(pay_profile_id, rate_dimension)` — mỗi profile chỉ có 1 config mỗi dimension tại 1 thời điểm. |
| **BR-PR-PRC02** | `rate_dimension = REGULAR` bắt buộc phải có nếu profile có `pay_method = HOURLY`. Đây là base rate để tính các dimension khác khi `rate_type = MULTIPLIER`. |
| **BR-PR-PRC03** | Khi `rate_type = MULTIPLIER` và `statutory_rule_id ≠ null` → engine **ưu tiên** lấy multiplier từ statutory_rule (pháp định). `base_rate` giữ vai trò fallback nếu statutory_rule không trả về giá trị. |
| **BR-PR-PRC04** | OT dimensions (`OT_WEEKDAY`, `OT_WEEKEND`, `OT_HOLIDAY`) thường dùng `rate_type = MULTIPLIER` với `statutory_rule_id → VN_OT_MULT_2019`. Không hard-code 1.5/2.0/3.0 trong base_rate. |
| **BR-PR-PRC05** | `HAZARDOUS` rate cộng thêm vào REGULAR rate (additive), không thay thế. Engine: `total_rate = regular_rate + hazardous_premium`. Documented trong formula. |
| **BR-PR-PRC06** | Worker có `worker_rate_override` cho cùng dimension → override wins, profile rate bị bỏ qua. Engine không cộng cả 2. |

## 4. Quan hệ với các entity khác

```
pay_profile_rate_config
├── pay_profile_id  → pay_master.pay_profile    (profile nào)
├── statutory_rule_id → pay_master.statutory_rule (OT multiplier)
└── overridden by: worker_rate_override          (per-worker exception)

Downstream:
└── pay_engine:  rate lookup per (employee × rate_dimension × period)
```

## 6. Query Patterns thường gặp

```sql
-- Lấy tất cả rate configs của 1 profile
SELECT rate_dimension, rate_type, base_rate, sr.code AS statutory_code
FROM pay_master.pay_profile_rate_config prc
LEFT JOIN pay_master.statutory_rule sr ON sr.id = prc.statutory_rule_id
WHERE prc.pay_profile_id = :profile_id
  AND prc.is_active = TRUE
  AND (prc.effective_end_date IS NULL OR prc.effective_end_date >= :payroll_date)
ORDER BY rate_dimension;

-- Engine lookup: rate cho (employee, dimension) — 3-layer resolution
WITH override AS (
  SELECT base_rate, statutory_rule_id
  FROM pay_master.worker_rate_override
  WHERE employee_id = :emp_id AND rate_dimension = :dimension
    AND is_active = TRUE LIMIT 1
),
profile_rate AS (
  SELECT prc.base_rate, prc.statutory_rule_id
  FROM pay_master.pay_profile_map ppm
  JOIN pay_master.pay_profile_rate_config prc ON prc.pay_profile_id = ppm.pay_profile_id
  WHERE ppm.employee_id = :emp_id AND prc.rate_dimension = :dimension
    AND prc.is_active = TRUE LIMIT 1
)
SELECT COALESCE(o.base_rate, p.base_rate) AS resolved_rate
FROM (SELECT 1) dummy
LEFT JOIN override o ON TRUE
LEFT JOIN profile_rate p ON TRUE;
```

## 7. Design Notes

> [!IMPORTANT]
> **Không hard-code OT multipliers.** Luôn link `statutory_rule_id → VN_OT_MULT_2019` cho VN OT dimensions. Khi luật thay đổi, chỉ update statutory_rule một chỗ — tất cả profiles tự áp dụng.

> [!NOTE]
> **NIGHT rate không phải OT:** Night shift premium (phụ cấp ca đêm 30% — BLLĐ Điều 98 khoản 2) là additive trên REGULAR rate, không phải OT. `OT_WEEKDAY` rate có thể cộng thêm night premium nếu worker làm OT ca đêm — engine phải handle composite rate.
