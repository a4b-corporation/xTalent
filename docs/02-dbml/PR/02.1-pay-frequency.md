# pay_frequency — Tần suất Trả lương (Pay Frequency)

> **Schema:** `pay_master.pay_frequency`
> **DDD Classification:** Reference (lookup table, PK = code)
> **SCD-2:** Không — reference data tĩnh, không versioned

---

## 1. Config những gì?

`pay_frequency` là lookup table định nghĩa các chu kỳ trả lương hỗ trợ. Đây là reference data cơ bản nhất của pay_master — mọi `pay_calendar` đều phải chọn 1 frequency.

> **Đặc điểm:** PK là `code` (không phải uuid) — reference data dạng enum có thêm metadata (`period_days`). Không cần SCD-2 vì frequency là khái niệm cố định.

### Fields

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `code` | varchar(20) **PK** | Mã tần suất | `MONTHLY`, `BIWEEKLY`, `WEEKLY`, `SEMI_MONTHLY` |
| `name` | varchar(50) | Tên hiển thị | `Hàng tháng`, `2 tuần/lần`, `Hàng tuần` |
| `period_days` | smallint | Số ngày mỗi kỳ lương | `30`, `14`, `7`, `15` |

---

## 2. Enum Values — Tần suất phổ biến

| code | name | period_days | Phổ biến tại |
|------|------|:-----------:|-------------|
| `MONTHLY` | Hàng tháng | 30 | VN (phổ biến nhất), hầu hết châu Á |
| `SEMI_MONTHLY` | 2 lần/tháng | 15 | Philippines, một số MNC tại VN |
| `BIWEEKLY` | 2 tuần/lần | 14 | Mỹ, Úc, Canada |
| `WEEKLY` | Hàng tuần | 7 | Ngành xây dựng, thời vụ |
| `DAILY` | Hàng ngày | 1 | Lao động thời vụ, daily workers |
| `QUARTERLY` | Hàng quý | 91 | Một số khoản thưởng, commissions |
| `ANNUAL` | Hàng năm | 365 | Thưởng năm, long-term incentives |

> **Lưu ý VN context:** >95% doanh nghiệp VN dùng `MONTHLY`. `SEMI_MONTHLY` và `BIWEEKLY` thường gặp ở công ty FDI, MNC.

---

## 3. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-FQ01** | `period_days` chỉ mang tính tham khảo. Ngày trả lương thực tế do `pay_calendar.calendar_json` quyết định (cut-off date, pay date). |
| **BR-PR-FQ02** | Không thể xóa frequency đang được `pay_calendar` reference. Phải update tất cả calendar trước. |
| **BR-PR-FQ03** | `balance_def.reset_freq_code` cũng reference table này — tần suất reset balance YTD/QTD. |

---

## 4. Quan hệ với các entity khác

```
pay_frequency (code PK)
    ↑ referenced by:
    ├── pay_calendar.frequency_code   → lịch lương dùng tần suất này
    └── balance_def.reset_freq_code   → balance reset theo tần suất
```

---

## 6. Design Notes

> [!NOTE]
> **PK = code (không phải uuid):** `pay_frequency` dùng natural key như `MONTHLY`, `BIWEEKLY` thay vì UUID vì đây là reference data có ý nghĩa cho người đọc. FK từ `pay_calendar.frequency_code` reference trực tiếp vào code string này.
