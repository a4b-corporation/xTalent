# piece_rate_config — Cấu hình Lương Sản phẩm (Piece Rate)

> **Schema:** `pay_master.piece_rate_config`
> **DDD Classification:** Value Object
> **SCD-2:** Không — versioned qua `effective_start_date / effective_end_date`
> **Changed:** 27Mar2026 (NEW — AQ-13 Option C)

---

## 1. Config những gì?

`piece_rate_config` định nghĩa đơn giá lương theo sản phẩm (lương khoán): **tổ hợp (product_code × quality_grade)** → đơn giá mỗi đơn vị. Phù hợp với ngành dệt may, lắp ráp điện tử, chế biến thực phẩm.

> **Data flow:** TA/Production → ghi số lượng sản phẩm → Engine lookup `piece_rate_config` → tính pay.
> `piece_rate_config` áp dụng cho `pay_profile` có `pay_method = PIECE_RATE`.

### Nhóm 1 — Định danh sản phẩm

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `pay_profile_id` | uuid FK | Profile áp dụng. `NULL` = global rate (mọi PIECE_RATE profiles) | FK → `pay_master.pay_profile` |
| `product_code` | varchar(50) NOT NULL | Mã sản phẩm | `SHIRT_BASIC`, `PCB_BOARD_A1`, `SHOE_SIZE_40` |
| `product_name` | varchar(100) | Tên sản phẩm | `Áo sơ mi cơ bản`, `Bo mạch PCB loại A1` |
| `quality_grade` | varchar(20) NOT NULL | Cấp chất lượng | `STANDARD`, `GRADE_A`, `GRADE_B`, `GRADE_C`, `REJECT` |
| `unit_code` | varchar(20) NOT NULL | Đơn vị tính | `PIECE`, `KG`, `METER`, `PAIR`, `SET` |

### Nhóm 2 — Đơn giá

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `rate_per_unit` | decimal(18,4) NOT NULL | Đơn giá mỗi đơn vị (VND) | `5000.0000` (VND/cái) |
| `quality_multiplier` | decimal(8,4) | Hệ số chất lượng. `GRADE_A = 1.1 × STANDARD rate` | `1.10`, `0.50` (REJECT) |
| `min_qty_per_period` | decimal(10,2) | Sản lượng tối thiểu để nhận rate này | `500` (cái/tháng) |
| `max_qty_per_period` | decimal(10,2) | Sản lượng tối đa nhận rate này (bậc thang) | `1000` |

### Nhóm 3 — Statutory Links

| Field | Type | Ý nghĩa |
|-------|------|---------|
| `min_wage_rule_id` | uuid FK | Rule kiểm tra tổng lương tháng ≥ lương tối thiểu vùng | FK → `pay_master.statutory_rule` |
| `ot_multiplier_rule_id` | uuid FK | Rule OT nếu sản lượng OT được tính riêng | FK → `pay_master.statutory_rule` |

### Nhóm 4 — Thời gian

| Field | Type | Ý nghĩa |
|-------|------|---------|
| `effective_start_date` | date | |
| `effective_end_date` | date | `NULL` = đang active |
| `is_active` | boolean | |

## 2. Enum Values

### `quality_grade`

| Giá trị | Ý nghĩa | Multiplier typical |
|---------|---------|:-----------------:|
| `STANDARD` | Đạt tiêu chuẩn xuất xưởng | 1.00 (base) |
| `GRADE_A` | Chất lượng xuất khẩu cao cấp | 1.10–1.20× |
| `GRADE_B` | Chất lượng nội địa | 0.90–0.95× |
| `GRADE_C` | Sản phẩm lỗi nhẹ — bán giá rẻ | 0.70–0.80× |
| `REJECT` | Hàng lỗi — không tính lương hoặc phạt | 0.00–0.30× |

### `unit_code`

| Giá trị | Ngành áp dụng |
|---------|--------------|
| `PIECE` | May mặc, lắp ráp điện tử, đóng gói |
| `KG` | Chế biến thực phẩm, thu hoạch nông sản |
| `METER` | Dệt vải, sản xuất vật liệu cuộn |
| `PAIR` | Giày dép |
| `SET` | Sản phẩm nhiều chi tiết (bàn + ghế, bộ đồ...) |

## 3. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-PRC01** | Unique: `(pay_profile_id, product_code, quality_grade)` per active period. Không được config 2 rates cho cùng (profile, product, grade). |
| **BR-PR-PRC02** | `pay_profile_id = NULL` → global rate, dùng cho mọi PIECE_RATE profile trừ khi profile đó có config riêng. Profile-specific config wins over global. |
| **BR-PR-PRC03** | `min_wage_rule_id` bắt buộc đối với VN PIECE_RATE profiles (BLLĐ Điều 91 + Điều 100 — lương khoán không được thấp hơn lương tối thiểu vùng × số công chuẩn). Engine phải kiểm tra sau khi tính total piece pay. |
| **BR-PR-PRC04** | `REJECT` grade: tùy chính sách công ty có thể rate = 0 (không tính), hoặc rate thấp (vẫn trả nhưng ít), hoặc phạt (trừ vào lương). Config trong `quality_multiplier`. |
| **BR-PR-PRC05** | Bậc thang sản lượng: nếu cần tier pricing (100–500 cái = 5k, 501–1000 = 5.5k), tạo 2 records với `min_qty / max_qty` khác nhau thay vì 1 record. Engine đọc tier phù hợp. |
| **BR-PR-PRC06** | **Min wage check (cuối kỳ):** `IF total_piece_pay < min_wage_region × actual_work_days / standard_days → pay = min_wage_substitute; flag PIECE_RATE_SUPPLEMENT`. Supplement amount = difference. |

## 4. Quan hệ với các entity khác

```
piece_rate_config
├── pay_profile_id     → pay_master.pay_profile    (PIECE_RATE profiles only)
├── min_wage_rule_id   → pay_master.statutory_rule  (VN_MIN_WAGE_2025)
├── ot_multiplier_rule_id → pay_master.statutory_rule (VN_OT_MULT_2019)

Input source:
└── ta.input_source_config → mang quantity_produced per (employee, product, date, grade)

Downstream:
└── pay_engine.result → PIECE_RATE_PAY element computed
```

## 6. Query Patterns thường gặp

```sql
-- Rate lookup cho (product, grade) của profile
SELECT rate_per_unit, quality_multiplier, unit_code,
       rate_per_unit * quality_multiplier AS effective_rate
FROM pay_master.piece_rate_config
WHERE (pay_profile_id = :profile_id OR pay_profile_id IS NULL)
  AND product_code = :product_code
  AND quality_grade = :quality_grade
  AND :qty BETWEEN COALESCE(min_qty_per_period, 0) AND COALESCE(max_qty_per_period, 999999)
  AND is_active = TRUE
  AND (effective_end_date IS NULL OR effective_end_date >= :payroll_date)
ORDER BY pay_profile_id NULLS LAST  -- profile-specific wins over global
LIMIT 1;

-- Tổng lương sản phẩm cho tháng — và min wage check
SELECT
  employee_id,
  SUM(qty_produced * effective_rate) AS total_piece_pay,
  (SELECT rate_per_unit FROM pay_master.piece_rate_config WHERE ...) AS min_wage
FROM production_input_log
JOIN piece_rate_config ON ...
GROUP BY employee_id
HAVING SUM(qty_produced * effective_rate) < (SELECT min_wage_amount FROM ...) -- flag supplement
```

## 7. Design Notes

> [!IMPORTANT]
> **Min wage supplement là bắt buộc (BLLĐ Điều 91, 100).** Lương khoán không được thấp hơn lương tối thiểu vùng × tỷ lệ ngày công thực tế. Engine phải tự động tính và add supplement, không để HR làm thủ công. `PIECE_RATE_SUPPLEMENT` element phải có trong profile nếu `pay_method = PIECE_RATE`.

> [!NOTE]
> **`pay_profile_id = NULL` (global rate) vs profile-specific:** Global rate dùng cho sản phẩm đơn giản có rate đồng nhất. Profile-specific cho phép nhà máy có 2 chuyền (chuyền XK rate cao hơn chuyền nội địa) dùng cùng product_code nhưng rate khác. Application phải resolve: profile-specific first.
