# validation_rule — Quy tắc Kiểm tra Lương (Payroll Validation Rule)

> **Schema:** `pay_master.validation_rule`
> **DDD Classification:** Value Object
> **Changed:** JUL 2025 (initial)

---

## 1. Config những gì?

`validation_rule` định nghĩa các quy tắc kiểm tra dữ liệu và kết quả trong payroll run — từ input validation (trước khi tính) đến result validation (sau khi tính). Engine execute tất cả active rules trước khi confirm run.

> **Vai trò của PR validation:** Light-check sau tính toán. OT cap enforcement chính = TA module. PR chỉ secondary-check nếu TA missed.

### Fields

| Field | Type | Ý nghĩa | Ví dụ |
|-------|------|---------|-------|
| `code` | varchar(50) UNIQUE | Mã rule | `VAL_PIT_POSITIVE`, `VAL_OT_CAP_CHECK` |
| `name` | varchar(100) | Tên | `PIT không âm`, `OT cap 300h/năm (secondary)` |
| `rule_type` | varchar(30) NOT NULL | Loại kiểm tra | Xem enum |
| `severity` | varchar(10) NOT NULL | Mức nghiêm trọng | `ERROR` (block), `WARNING` (alert only) |
| `element_id` | uuid FK | Element áp dụng. `NULL` = toàn bộ run | FK → `pay_master.pay_element` |
| `condition_json` | jsonb | Điều kiện kiểm tra | `{"operator": ">=", "value": 0}` |
| `error_message` | varchar(500) | Thông báo lỗi | `"PIT amount < 0 — check taxable income"` |
| `is_active` | boolean | | `true` |

---

## 2. Enum — `rule_type` & `severity`

### `rule_type`
| Giá trị | Kiểm tra | Phase |
|---------|---------|-------|
| `INPUT_CHECK` | Validate inputs trước khi tính | Pre-calculation |
| `RESULT_CHECK` | Validate kết quả sau tính | Post-calculation |
| `CROSS_MODULE` | Check dữ liệu từ TA/absence | Pre-calculation |
| `STATUTORY_CAP` | Kiểm tra trần/sàn pháp định | Post-calculation |

### `severity`
| Giá trị | Hành vi |
|---------|---------|
| `ERROR` | Block run — không thể confirm cho đến khi fix |
| `WARNING` | Thông báo cho payroll operator, vẫn cho phép confirm |

---

## 3. Business Rules

| BR | Mô tả |
|----|-------|
| **BR-PR-VR01** | `severity = ERROR` → run không thể move sang `CONFIRMED` status. Operator phải review và fix trước. |
| **BR-PR-VR02** | `severity = WARNING` → ghi vào `pay_engine.calc_log`, operator thấy trong review screen. Có thể override manually. |
| **BR-PR-VR03** | OT cap (`VAL_OT_CAP_*`) → `severity = WARNING` (không phải ERROR) vì TA đã primary-enforce. PR chỉ cảnh báo nếu số liệu OT vượt ngưỡng. |

## 5. Design Notes

> [!NOTE]
> **OT cap: TA là primary, PR là secondary.** `VAL_OT_ANNUAL_CAP` dùng `severity = WARNING` because TA already blocked excess OT at timesheet approval. PR fires warning as a safety net — nếu thấy warning này thường xuyên, investigate TA config.

> [!NOTE]
> **`condition_json` không có DB schema.** Application layer validate structure trước khi save. Engine interpret `condition_json` field theo `rule_type`.
